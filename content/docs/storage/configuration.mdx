---
title: Storage configuration
description: How the starter chooses local filesystem vs S3-compatible storage and which environment variables apply.
---

Storage is implemented with [unstorage](https://unstorage.unjs.io/) in `shared/lib/storage/storage.ts`. The app picks **filesystem** storage by default, or **S3-compatible** storage when all required S3 variables are set.

## Selection logic

`getFileStorage()` resolves once per process: if every S3 variable is present, the S3 driver is used; otherwise the filesystem driver is used with a local directory under the project root.

```ts title="shared/lib/storage/storage.ts"
function shouldUseS3Storage() {
  return Boolean(
    process.env.STORAGE_S3_ENDPOINT &&
    process.env.STORAGE_S3_BUCKET &&
    process.env.STORAGE_S3_ACCESS_KEY_ID &&
    process.env.STORAGE_S3_SECRET_ACCESS_KEY &&
    process.env.STORAGE_S3_REGION,
  );
}
```

> **Note:** These variables are not listed in `.env.example`; add them to your `.env` when you enable S3. If any one is missing, the app falls back to filesystem storage.

## Filesystem (default)

On first use, the code ensures a `.storage` directory exists at the project working directory and mounts the FS driver there:

```ts title="shared/lib/storage/storage.ts"
async function ensureLocalStorageBase() {
  const basePath = path.join(process.cwd(), ".storage");
  await mkdir(basePath, { recursive: true });
  return basePath;
}
```

```ts title="shared/lib/storage/storage.ts"
async function createFsStorage() {
  const base = await ensureLocalStorageBase();

  return createStorage({
    driver: fsDriver({ base }),
  });
}
```

Treat `.storage` as **ephemeral on serverless hosts** unless you mount persistent storage or switch to S3.

## S3-compatible storage

When `shouldUseS3Storage()` is true, unstorage’s S3 driver is configured with:

| Variable | Role |
| -------- | ---- |
| `STORAGE_S3_ENDPOINT` | API endpoint URL (works with AWS S3, MinIO, R2, etc.) |
| `STORAGE_S3_BUCKET` | Bucket name |
| `STORAGE_S3_ACCESS_KEY_ID` | Access key |
| `STORAGE_S3_SECRET_ACCESS_KEY` | Secret key |
| `STORAGE_S3_REGION` | Region string required by the driver |

```ts title="shared/lib/storage/storage.ts"
function createS3Storage() {
  return createStorage({
    driver: s3Driver({
      endpoint: process.env.STORAGE_S3_ENDPOINT!,
      bucket: process.env.STORAGE_S3_BUCKET!,
      accessKeyId: process.env.STORAGE_S3_ACCESS_KEY_ID!,
      secretAccessKey: process.env.STORAGE_S3_SECRET_ACCESS_KEY!,
      region: process.env.STORAGE_S3_REGION!,
    }),
  });
}
```

## Singleton

`getFileStorage()` caches the resolved storage instance for the lifetime of the Node process:

```ts title="shared/lib/storage/storage.ts"
let storagePromise: Promise<ReturnType<typeof createS3Storage>> | null = null;

export function getFileStorage() {
  if (!storagePromise) {
    storagePromise = Promise.resolve(
      shouldUseS3Storage() ? createS3Storage() : createFsStorage(),
    );
  }

  return storagePromise;
}
```

Changing env vars at runtime does not recreate storage until the process restarts.
