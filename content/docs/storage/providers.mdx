---
title: Storage providers
description: Filesystem default storage and S3-compatible backends as wired in storage.ts.
---

Provider behavior is defined entirely in `shared/lib/storage/storage.ts`. There is no separate plugin registry—only two drivers are used: **filesystem** and **S3** (via unstorage).

## Filesystem (default)

Used when **any** of the S3 environment variables is missing. The driver writes under:

- **Base path:** `{process.cwd()}/.storage`
- **Creation:** `mkdir(..., { recursive: true })` runs before the driver is created.

This matches local development and single-node deployments with a writable disk. On platforms without a persistent filesystem (typical serverless), files may disappear between invocations unless you use external storage.

> **Note:** Do not commit `.storage` to version control if it contains real uploads; add it to `.gitignore` if you use local storage in dev.

## S3-compatible (unstorage S3 driver)

When **all** of these are set, the S3 driver is used:

- `STORAGE_S3_ENDPOINT`
- `STORAGE_S3_BUCKET`
- `STORAGE_S3_ACCESS_KEY_ID`
- `STORAGE_S3_SECRET_ACCESS_KEY`
- `STORAGE_S3_REGION`

The configuration passed to unstorage is:

```ts title="shared/lib/storage/storage.ts"
function createS3Storage() {
  return createStorage({
    driver: s3Driver({
      endpoint: process.env.STORAGE_S3_ENDPOINT!,
      bucket: process.env.STORAGE_S3_BUCKET!,
      accessKeyId: process.env.STORAGE_S3_ACCESS_KEY_ID!,
      secretAccessKey: process.env.STORAGE_S3_SECRET_ACCESS_KEY!,
      region: process.env.STORAGE_S3_REGION!,
    }),
  });
}
```

Compatible services include **Amazon S3**, **Cloudflare R2**, **MinIO**, and other S3 API–compatible endpoints. Set `STORAGE_S3_ENDPOINT` to your vendor’s API URL and choose a region string your provider accepts.

> **Note:** The starter does not configure public bucket URLs or CDN caching for raw keys; downloads go through the authenticated app route described in [Bucket access](./bucket-access.mdx).

## Switching providers

1. Provision a bucket and credentials.
2. Set all five `STORAGE_S3_*` variables and restart the app.
3. Existing rows in `StoredFile` point at keys in the **previous** backend; migrating data between filesystem and S3 (or between buckets) is a manual or scripted process outside this repo.
