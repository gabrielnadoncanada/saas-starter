title: Using storage in code
description: Upload, read, and delete files via the storage service and how plan limits apply.
---

Server-side helpers live in `shared/lib/storage/storage-service.ts` (marked `server-only`). They coordinate **unstorage** (`getFileStorage()` from `shared/lib/storage/storage.ts`), the **Prisma** `StoredFile` model, and **billing entitlements** for per-organization storage limits.

## Keys and metadata

Object keys are generated per upload: `{organizationId}/{uuid}-{sanitized-name}{ext}`. The database row stores `storageKey`, original `fileName`, `contentType`, `sizeBytes`, and relations to organization and uploader.

## Upload (`saveStoredFile`)

Pass the active organization id, the uploading user id, and a Web `File` (for example from a form in a Server Action):

```ts title="shared/lib/storage/storage-service.ts"
export async function saveStoredFile(input: {
  organizationId: string;
  uploadedByUserId: string;
  file: File;
}) {
  const entitlements = await getCurrentOrganizationEntitlements();

  if (!entitlements || entitlements.organizationId !== input.organizationId) {
    throw new Error("Unable to determine organization plan");
  }

  const sizeBytes = input.file.size;
  const currentUsage = await getOrganizationStorageUsage(input.organizationId);
  const limitBytes = getPlanLimit(entitlements, "storageMb") * 1024 * 1024;

  if (currentUsage + sizeBytes > limitBytes) {
    throw new Error("Storage limit reached for the current plan.");
  }

  const storageKey = buildStorageKey(input.organizationId, input.file.name);
  const storage = await getFileStorage();
  const buffer = Buffer.from(await input.file.arrayBuffer());

  await storage.setItemRaw(storageKey, buffer);

  return db.storedFile.create({
    data: {
      organizationId: input.organizationId,
      uploadedByUserId: input.uploadedByUserId,
      storageKey,
      fileName: input.file.name,
      contentType: input.file.type || "application/octet-stream",
      sizeBytes,
    },
  });
}
```

> **Note:** Uploads require matching entitlements for `input.organizationId`. Total stored bytes for the org cannot exceed the plan’s `storageMb` limit (see `shared/config/billing.config.ts`).

## Read bytes (`readStoredFileBody`)

Use the **storage key** from a `StoredFile` row (not the public URL alone):

```ts title="shared/lib/storage/storage-service.ts"
export async function readStoredFileBody(storageKey: string) {
  const storage = await getFileStorage();
  const body = await storage.getItemRaw(storageKey);

  if (!body) {
    throw new Error("Stored file is missing from backing storage.");
  }

  if (body instanceof Uint8Array) {
    return body;
  }

  if (typeof body === "string") {
    return Buffer.from(body);
  }

  return Buffer.from(body as ArrayBuffer);
}
```

## Look up a row (`getStoredFileRecord`)

Restricts by both `storedFileId` and `organizationId` so callers cannot cross organizations:

```ts title="shared/lib/storage/storage-service.ts"
export async function getStoredFileRecord(
  storedFileId: string,
  organizationId: string,
) {
  return db.storedFile.findFirst({
    where: {
      id: storedFileId,
      organizationId,
    },
  });
}
```

## Delete (`deleteStoredFile`)

Removes the blob from backing storage, then deletes the database row:

```ts title="shared/lib/storage/storage-service.ts"
export async function deleteStoredFile(
  storedFileId: string,
  organizationId: string,
) {
  const storedFile = await getStoredFileRecord(storedFileId, organizationId);

  if (!storedFile) {
    throw new Error("File not found");
  }

  const storage = await getFileStorage();
  await storage.removeItem(storedFile.storageKey);

  await db.storedFile.delete({
    where: { id: storedFile.id },
  });
}
```

## Usage totals

`getOrganizationStorageUsage(organizationId)` sums `sizeBytes` for all `StoredFile` rows in that organization and is used when enforcing limits on upload.
