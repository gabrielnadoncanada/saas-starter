---
title: File access and serving
description: How GET /api/files/[storedFileId] authorizes downloads and streams file bytes.
---

The only HTTP surface for stored files in this starter is **`GET /api/files/[storedFileId]`** (`app/api/files/[storedFileId]/route.ts`). There is no anonymous public URL to objects in `.storage` or S3; access is **session + organization scoped**.

## Authorization

The handler calls `requireActiveOrganizationMembership()` from `features/organizations/server/organization-membership.ts`. That ensures the user is signed in and has an **active organization** membership. The stored file is then loaded only if its `organizationId` matches that membership:

```ts title="app/api/files/[storedFileId]/route.ts"
export async function GET(
  _request: Request,
  context: { params: Promise<{ storedFileId: string }> },
) {
  try {
    const membership = await requireActiveOrganizationMembership();
    const { storedFileId } = await context.params;
    const storedFile = await getStoredFileRecord(
      storedFileId,
      membership.organizationId,
    );

    if (!storedFile) {
      return NextResponse.json({ error: "File not found" }, { status: 404 });
    }
```

So:

- Unauthenticated users get an error response (typically **400** from the catch block when the guard throws).
- Authenticated users **not** in the file’s organization receive **404** (no row match), not a leak of another org’s metadata.

## Response body and headers

The route reads bytes via `readStoredFileBody(storedFile.storageKey)` and returns them as a `Blob` with the stored `contentType`. Headers include private caching, attachment disposition, length, and type:

```ts title="app/api/files/[storedFileId]/route.ts"
    const body = await readStoredFileBody(storedFile.storageKey);
    const arrayBuffer = body.buffer.slice(
      body.byteOffset,
      body.byteOffset + body.byteLength,
    ) as ArrayBuffer;
    const blob = new Blob([arrayBuffer], { type: storedFile.contentType });

    return new NextResponse(blob, {
      headers: {
        "Cache-Control": "private, max-age=0, must-revalidate",
        "Content-Disposition": `attachment; filename="${storedFile.fileName}"`,
        "Content-Length": String(storedFile.sizeBytes),
        "Content-Type": storedFile.contentType,
      },
    });
```

> **Note:** `Content-Disposition` is set to **attachment**, so browsers treat the response as a download with the original filename.

## Errors

Failures (including membership errors) are returned as JSON with status **400** and a message; missing files for the org return **404**.

```ts title="app/api/files/[storedFileId]/route.ts"
  } catch (error) {
    return NextResponse.json(
      {
        error:
          error instanceof Error ? error.message : "Unable to download file",
      },
      { status: 400 },
    );
  }
```

## Database model

`StoredFile` ties each object to one organization and uploader; see `prisma/models/platform.prisma`. There is no per-file sharing flag in the schema—access is organization-wide for members who pass the route’s checks.
