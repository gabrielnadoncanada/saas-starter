## Plan configuration

Plans use `pricingModel: "per_seat"` when billing scales with **seat quantity**. In the default catalog, the **Team** plan uses `kind: "seat"` line items for monthly and yearly schedules (when `STRIPE_PRICE_TEAM_*` env vars are set).

```ts title="shared/config/billing.config.ts"
    {
      id: "team",
      name: "Team",
      description: "For teams that need seats, analytics, and higher limits.",
      pricingModel: "per_seat",
      features: [
        "Per-seat billing",
        "Team analytics",
        "Higher usage limits",
        "AI assistant",
      ],
      // ...
      schedules: {
        month: process.env.STRIPE_PRICE_TEAM_MONTHLY
          ? {
              lineItems: [
                {
                  id: "seat",
                  kind: "seat",
                  price: price(
                    process.env.STRIPE_PRICE_TEAM_MONTHLY,
                    4900,
                    7,
                  )!,
                  unitLabel: "seat",
                },
              ],
            }
          : undefined,
```

## Checkout and subscription updates

- **`buildPlanCheckoutLineItems`** maps each recurring line to Stripe `{ price, quantity }` using **`buildRecurringSelectionItems`**, which applies seat math for `kind === "seat"` (see Advanced pricing).
- **`createOrganizationSubscriptionCheckoutSession`** passes `seatQuantity` from the action and stores it in session/subscription metadata.
- **`updateOrganizationSubscriptionConfiguration`** rebuilds Stripe subscription items from the same selection helper when the owner changes plan, interval, or seats.

## Billing settings UI

`BillingPlanSelector` shows a numeric **Seat quantity** input when `selectedPlan.pricingModel === "per_seat"`, with minimum 1, and posts `seatQuantity` as a hidden field with plan id and interval.

```tsx title="features/billing/components/billing-plan-selector.tsx"
      {selectedPlan.pricingModel === "per_seat" ? (
        <label className="block space-y-2">
          <span className="text-sm font-medium">Seat quantity</span>
          <Input
            min={1}
            type="number"
            value={seatQuantity}
            onChange={(event) =>
              setSeatQuantity(Math.max(1, Number(event.target.value) || 1))
            }
          />
        </label>
      ) : null}
```

Default seat quantity on the billing page comes from **`entitlements.seats`** or falls back to **current member count** when seats are not set.

> **Note:** A `per_seat` plan can still include **`includedQuantity`** on a seat line item to offer free seats before billing kicks in (see `buildRecurringSelectionItems`).
