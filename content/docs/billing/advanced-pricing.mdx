## What "advanced" means here

The starter does **not** configure Stripe graduated/tiered price tiers in code. Instead, it models:

- **Per-interval pricing** (`month` / `year`) with one or more **line items**.
- Each line item has a **`kind`** of `"flat"` (quantity 1 in checkout).

```ts title="shared/config/billing.config.ts"
export type BillingRecurringLineItem = {
  component: string;
  kind: BillingLineItemKind;
  price: BillingPrice;
  unitLabel?: string;
};

export type BillingIntervalPricing = {
  lineItems: readonly BillingRecurringLineItem[];
};
```

## Building checkout quantities

`buildLineItemsWithQuantities` maps each line item to a Stripe line with quantity `1`.

```ts title="features/billing/catalog.ts"
  return intervalPricing.lineItems.map((lineItem) => ({
    componentKey: lineItem.component,
    itemKey: plan.id,
    itemType: "plan" as const,
    priceId: lineItem.price.priceId,
    quantity: 1,
  }));
```

## Catalog index for webhooks

`catalog.ts` indexes all configured recurring prices so webhook sync can resolve Stripe price ids back to plan metadata via `findCatalogPrice`.

> **Note:** To model multiple recurring components per plan (e.g. base + add-on), add multiple entries to `lineItems`.
