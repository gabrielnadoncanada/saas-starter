## File location

All product semantics for the app live in **`shared/config/billing.config.ts`** (imported as `server-only` consumers should use `features/billing/catalog.ts` for reads).

## Editing workflow (from source comments)

1. Create products and prices in the Stripe Dashboard.
2. Copy each price id into the matching `STRIPE_PRICE_*` variable in `.env`.
3. Edit the `plans` array: names, features, capabilities, limits.
4. Capabilities gate features (for example `ai.assistant`); limits gate quotas (`tasksPerMonth`, `teamMembers`, `storageMb`, `aiCredits`).

## Core types

- **`capabilities`** — Closed list of strings such as `task.create`, `team.invite`, `ai.assistant`, `billing.portal`.
- **`limitKeys`** — `tasksPerMonth`, `teamMembers`, `storageMb`, `aiCredits`.
- **`BillingPlan`** — `id`, `name`, `description`, optional `highlighted`, `features` (marketing strings), `capabilities`, `limits`, and **`intervalPricing`** per billing interval (`month` / `year`).

```ts title="shared/config/billing.config.ts"
export type BillingPlan = {
  id: string;
  name: string;
  description: string;
  highlighted?: boolean;
  features: string[];
  capabilities: Capability[];
  limits: Record<LimitKey, number>;
  intervalPricing: Partial<Record<BillingInterval, BillingIntervalPricing>>;
};
```

## Default catalog snapshot

- **free** — no Stripe interval pricing in-repo; limits e.g. 10 tasks/month, 1 team member, 100 MB storage; capabilities `task.create`, `billing.portal`.
- **pro** — monthly/yearly line items when env price ids exist; higher limits and more capabilities including `ai.assistant`.
- **team** — monthly/yearly line items when env price ids exist; highest default limits and `team.analytics`.

> **Warning:** Paid intervals only appear in the UI if `intervalPricing.month` / `intervalPricing.year` include line items whose `price.priceId` is set from env. Missing `STRIPE_PRICE_*` values omit that interval.
