## File location

All product semantics for the app live in **`shared/config/billing.config.ts`** (imported as `server-only` consumers should use `features/billing/catalog.ts` for reads).

## Editing workflow (from source comments)

1. Create products and prices in the Stripe Dashboard.
2. Copy each price id into the matching `STRIPE_PRICE_*` variable in `.env`.
3. Edit the `plans` array: names, features, capabilities, limits.
4. Capabilities gate features (for example `ai.assistant`); limits gate quotas (`tasksPerMonth`, `teamMembers`, `storageMb`).
5. One-time add-ons go in `oneTimeProducts`.

## Core types

- **`capabilities`** — Closed list of strings such as `task.create`, `team.invite`, `ai.assistant`, `billing.portal`.
- **`limitKeys`** — `tasksPerMonth`, `teamMembers`, `storageMb`.
- **`pricingModel`** — `"flat"` or `"per_seat"`.
- **`BillingPlanDefinition`** — `id`, `name`, `description`, optional `highlighted`, `pricingModel`, `features` (marketing strings), `capabilities`, `limits`, and **`schedules`** per billing interval (`month` / `year`).

## Default catalog snapshot

- **free** — `flat`, no Stripe schedules in-repo; limits e.g. 10 tasks/month, 1 team member, 100 MB storage; capabilities `task.create`, `billing.portal`.
- **pro** — `flat`, monthly/yearly line items when env price ids exist; higher limits and more capabilities including `ai.assistant`.
- **team** — `per_seat`, seat-priced line items when env price ids exist; highest default limits and `team.analytics`.

**One-time example:** `storage_boost` adds `limitEffect.storageMb` when purchased (see `oneTimeProducts` in the same file).

> **Warning:** Paid intervals only appear in the UI if `schedules.month` / `schedules.year` include line items whose `price.priceId` is set from env. Missing `STRIPE_PRICE_*` values omit that schedule.
