## Marketing pricing

`PricingSection` loads paid plans from `getPricingPlans()` (plans that have at least one configured Stripe-backed interval), enriches them with monthly/yearly display prices, sorts by monthly amount, and renders **`PricingToggle`** (client) for the public pricing UI.

```tsx title="features/billing/components/pricing-section.tsx"
export async function PricingSection() {
  const paidPlans = getPricingPlans()
    .map((plan) => ({
      planId: plan.id as PlanId,
      productName: plan.name,
      description: plan.description,
      highlighted: plan.highlighted ?? false,
      features: plan.features,
      monthly: getPlanDisplayPrice(plan.id, "month"),
      yearly: getPlanDisplayPrice(plan.id, "year"),
      sortAmount: getPlanDisplayPrice(plan.id, "month")?.unitAmount ?? 0,
    }))
    .filter((plan) => plan.monthly !== null)
    .sort((a, b) => a.sortAmount - b.sortAmount);

  return (
    <main className="mx-auto max-w-7xl px-4 py-12 sm:px-6 lg:px-8">
      <PricingToggle plans={paidPlans} />
    </main>
  );
}
```

## Settings → Billing

`/settings/billing` uses **`getBillingPageData()`** and renders **`BillingPlanSelector`** with flags derived from org context:

- **`canManageBilling`** — owner only; non-owners see a disabled “Only the owner can manage billing” button.
- **`canUpdateSubscription`** — owner, has subscription, and `stripeSubscriptionId` present (in-portal upgrades vs Checkout).
- **`canManagePortal`** — owner, subscription present, `stripeCustomerId` synced.

The selector shows interval tabs, plan radios, feature badges, primary submit (`startSubscriptionCheckoutAction` or `changePlanAction`), and **Open billing portal** (`openBillingPortalAction`).

## Cross-cutting widgets

- **`UpgradeCard`** — Generic CTA linking to `routes.marketing.pricing` (used on the dashboard when a capability is missing).
- **`UsageMeter`** — Progress bar for `current` vs `limit` with an 80% “near limit” tint.

Customize copy in **`billing.config.ts`** (`features`, `name`, `description`) and in these components; avoid hardcoding prices outside the catalog (display amounts come from `unitAmount` / Stripe).
