## What this is (and is not)

“Metered usage” in this starter means **application-enforced monthly quotas** stored in Prisma (`usageCounter`), not Stripe Billing metered prices. Stripe subscription status still determines which **plan limits** apply; consumption is tracked locally.

## Recording usage: `consumeMonthlyUsage`

For a given organization, limit key, and entitlements:

1. Computes **period start** as the first day of the current calendar month.
2. Reads the plan limit via `getPlanLimit(entitlements, limitKey)`.
3. `createMany` with `skipDuplicates` ensures a counter row exists for `(organizationId, limitKey, periodStart)`.
4. **`updateMany`** increments `count` only while `count < limit` (atomic guard).
5. If no row was updated, loads current usage and throws **`LimitReachedError`**.

```ts title="features/billing/server/usage-service.ts"
export async function consumeMonthlyUsage(
  organizationId: string,
  limitKey: LimitKey,
  entitlements: OrganizationEntitlements,
  deps: { db: UsageDbClient } = { db },
) {
  const periodStart = getPeriodStart();
  const limit = getPlanLimit(entitlements, limitKey);

  await deps.db.usageCounter.createMany({
    data: {
      organizationId,
      limitKey,
      periodStart,
      count: 0,
    },
    skipDuplicates: true,
  });

  const result = await deps.db.usageCounter.updateMany({
    where: {
      organizationId,
      limitKey,
      periodStart,
      count: {
        lt: limit,
      },
    },
    data: {
      count: {
        increment: 1,
      },
    },
  });

  if (result.count === 0) {
    const currentUsage = await getMonthlyUsage(organizationId, limitKey, deps);
    throw new LimitReachedError(
      limitKey,
      limit,
      currentUsage,
      entitlements.planName,
    );
  }
}
```

## Reading usage: `getMonthlyUsage`

Returns the counter’s `count` for the current month (or `0`), used by the dashboard and by guards that compare usage to limits.

## Example: tasks

`createTask` asserts `task.create`, then calls `consumeMonthlyUsage` with `"tasksPerMonth"` inside a transaction so task creation rolls back if the quota is exhausted.

> **Warning:** Adding new limit keys requires corresponding `limitKeys` entries in `billing.config.ts` and consistent use of `checkLimit` / `consumeMonthlyUsage` wherever the resource is created.
