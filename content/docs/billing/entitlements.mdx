title: Entitlements and feature gating
description: How organization entitlements are resolved from subscriptions and how capabilities gate features.
---

## `OrganizationEntitlements` shape

Resolved entitlements include plan id/name, billing interval, capabilities, limits, Stripe ids, and subscription status. See `OrganizationEntitlements` in `billing.config.ts`.

## Resolution (`resolveEntitlements`)

`features/billing/server/organization-entitlements.ts`:

1. Loads the latest **subscription** for the organization (`referenceId`).
2. If there is no subscription, or status fails `hasPlanAccess`, or plan id is unknown → **free defaults** via `getDefaultEntitlements`.
3. If subscribed → copies **capabilities** and **limits** from `getPlan(subscription.plan)`, attaches Stripe fields.

```ts title="features/billing/server/organization-entitlements.ts"
  const plan = getPlan(subscription.plan);

  return {
    ...base,
    billingInterval:
      subscription.billingInterval === "month" ||
      subscription.billingInterval === "year"
        ? subscription.billingInterval
        : null,
    capabilities: [...plan.capabilities],
    limits: { ...plan.limits },
    planId: plan.id as PlanId,
    planName: plan.name,
    stripeCustomerId: subscription.stripeCustomerId ?? null,
    stripeSubscriptionId: subscription.stripeSubscriptionId ?? null,
    subscriptionStatus: subscription.status ?? null,
  };
```

## Runtime guards (`plan-guards.ts`)

- **`hasCapability`** / **`assertCapability`** — Throws `UpgradeRequiredError` if the capability string is missing.
- **`getPlanLimit`**, **`checkLimit`**, **`assertLimit`** — Enforce numeric **limits** (tasks, members, storage).

Example: creating a task calls `assertCapability(entitlements, "task.create")` before consuming usage.

```ts title="features/billing/plan-guards.ts"
export function assertCapability(
  entitlements: OrganizationEntitlements,
  capability: Capability,
): void {
  if (!hasCapability(entitlements, capability)) {
    throw new UpgradeRequiredError(capability, entitlements.planName);
  }
}
```

> **Note:** The dashboard and billing UI also use `hasCapability` for conditional sections (for example hiding the AI card or upgrade prompts).
