## Layout

Tests live under **`test/`**, grouped by domain: `test/tasks/`, `test/billing/`, `test/auth/`, `test/organizations/`, etc. File names follow **`*.test.ts`** (and occasionally **`*.test.tsx`** for React).

## Mocking modules

Tests use **`vi.mock`** for heavy dependencies: Prisma (`db`), billing guards, `next/headers`, auth APIs, and `server-only`. Many suites use **`vi.hoisted`** so mock factories run before imports that consume them.

Example from task creation tests: mock Prisma and billing modules, then **dynamic import** the code under test so mocks apply first.

```ts title="test/tasks/create-task.test.ts"
const { db } = await import("@/shared/lib/db/prisma");
const { getCurrentOrganizationEntitlements } =
  await import("@/features/billing/server/organization-entitlements");
const { assertCapability } =
  await import("@/features/billing/plan-guards");
const { consumeMonthlyUsage } =
  await import("@/features/billing/server/usage-service");
const { createTask } = await import("@/features/tasks/server/task-mutations");
```

Organization action tests mock **`validatedAuthenticatedAction`** to pass the handler through directly, so the suite exercises the inner action logic with explicit “user” arguments.

```ts title="test/organizations/organization-owner.actions.test.ts"
vi.mock("@/shared/lib/auth/authenticated-action", () => ({
  validatedAuthenticatedAction: (_schema: unknown, action: unknown) => action,
}));
```

## Assertions

- Use **`expect`** for return values and **`vi.mocked(...).toHaveBeenCalledWith`** to assert collaborators received the right data.
- For errors, use **`expect(promise).rejects.toBeInstanceOf(ErrorClass)`** when testing failure paths.

## Component tests (React)

**`billing-plan-selector.test.tsx`** starts with:

```tsx title="test/billing/billing-plan-selector.test.tsx"
// @vitest-environment jsdom

import { render, screen } from "@testing-library/react";
import React from "react";
import { describe, expect, it, vi } from "vitest";
```

The **`@vitest-environment jsdom`** directive switches that file to a browser-like DOM. Child modules and server actions are mocked to keep the test focused on UI.

> **Note:** Align new tests with **`vitest.config.ts`** `include` patterns so `pnpm test` discovers them; add `**/*.test.tsx` to `include` if you rely on multiple TSX tests.
