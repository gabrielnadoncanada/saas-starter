## Where to find it

Personal account settings live at **`/settings`** (`routes.settings.profile`). The page is server-rendered and requires a signed-in user; otherwise it redirects to login.

## Profile card

The profile section shows avatar, name, and email. **Edit Profile** opens `EditProfileDialog`, a client dialog that posts to the `updateAccountAction` server action with:

- **Name** — required text field.
- **Profile image** — optional avatar upload.

```tsx title="features/account/components/settings/edit-profile-dialog.tsx"
        <form className="space-y-4" onSubmit={handleSubmit}>
          <Field>
            <FieldLabel>Profile image</FieldLabel>
            <CFileUpload
              defaultAvatar={image ?? undefined}
              onFileChange={setAvatar}
              className="items-start"
            />
            <FieldDescription>PNG, JPG, or WebP up to 2 MB.</FieldDescription>
          </Field>

          <Field data-invalid={nameField.invalid}>
            <FieldLabel htmlFor="name">Name</FieldLabel>
            <Input
              id="name"
              name="name"
              placeholder={"Enter your name"}
              defaultValue={nameField.value}
              aria-invalid={nameField.invalid}
              required
            />
            <FieldError>{nameField.error}</FieldError>
          </Field>
```

## Linked accounts

`LinkedAccountsCard` lists configured OAuth providers (from `getEnabledOAuthProviderIds()`). For each provider:

- **Link** — Calls Better Auth `link-social` with `disableRedirect: true`, then navigates to the returned URL.
- **Unlink** — Submits `unlinkAuthProviderAction` unless the provider is required (e.g. last sign-in method).

Query params `success`, `provider`, and `error` on `/settings` drive toast feedback for OAuth flows.

## Password

The password card uses `EditPasswordDialog` with `hasPassword` derived from linked-account overview:

- If the user already has a password: **Change password** — current password, new password, confirm (via `savePasswordAction` → Better Auth `changePassword`).
- If not: **Create a password** — new + confirm only (Better Auth `setPassword`).

## Delete account

**Delete Account** opens `DeleteAccountDialog`. The user must type the confirmation word from the account schema (`DELETE_CONFIRMATION_WORD`) to submit `deleteAccountAction`.

> **Warning:** Deletion is irreversible once the action succeeds; the UI states this explicitly.
