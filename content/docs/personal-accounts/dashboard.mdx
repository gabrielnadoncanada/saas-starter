## What the dashboard is

The app dashboard at `/dashboard` renders `DashboardOverview`, which loads organization context, billing entitlements, and workspace stats in one server pass via `getDashboardOverview()`.

```tsx title="app/(app)/dashboard/page.tsx"
import { DashboardOverview } from "@/features/dashboard/components/dashboard-overview";

export default async function DashboardPage() {
  return <DashboardOverview />;
}
```

## Summary cards

The overview shows:

- **Current plan** — Plan name, internal plan id badge, and display price (from the billing catalog) or “Free”.
- **Task creations this month** — Count for the calendar month with a `UsageMeter` for the monthly task quota (`tasksPerMonth`).
- **Organization members** — Member count for the active organization.
- **AI Assistant** — Shown only when the workspace has the `ai.assistant` capability; surfaces whether any AI conversations exist.

## Charts and onboarding

- **Task activity this week** — A bar chart (`DashboardUsageChart`) of tasks created in the last seven days (weekday labels).
- **Getting started** — `DashboardOnboardingChecklist` lists linked steps (first task, invite teammate, try assistant) with completion derived from live data; items are hidden when the plan does not allow them (for example `team.invite` or AI).

```ts title="features/dashboard/server/get-dashboard-overview.ts"
  const checklist = [
    {
      id: "first-task",
      title: "Create your first task",
      done: taskCount > 0,
      href: routes.app.tasks,
    },
    {
      id: "invite-team",
      title: "Invite a teammate",
      done: memberCount > 1,
      href: routes.settings.members,
      hidden: !entitlements || !hasCapability(entitlements, "team.invite"),
    },
    {
      id: "try-assistant",
      title: "Use the AI assistant",
      done: assistantConversationCount > 0,
      href: routes.app.assistant,
      hidden: !canUseAI,
    },
  ].filter((item) => !item.hidden);
```

## Recent tasks and upgrade prompt

- **Recent tasks** — Up to five latest tasks with title, code, status, and priority; links go to the tasks route.
- If the workspace lacks `team.analytics`, an `UpgradeCard` nudges users toward the marketing pricing page.

## Sidebar navigation

`DashboardSidebar` uses `dashboardSidebarData` (Dashboard + Tasks under “General”) and embeds the assistant nav in a collapsible group. The header shows `DashboardSidebarUser` (organization switcher, settings link, admin link for platform admins, sign out).

> **Note:** Dashboard copy assumes an organization-backed workspace; entitlements fall back to the free plan when the active org cannot be resolved.
