## Prerequisites

This project uses [shadcn/ui](https://ui.shadcn.com/) with the **New York** style, **TypeScript**, **React Server Components** enabled, and Tailwind CSS variables. Configuration is in the repo root `components.json`.

> **Note:** Aliases point UI code to `@/shared/components/ui` and shared utilities to `@/shared/lib/utils`, not to a top-level `components/` folder.

## Add a component

From the repository root, run:

```bash
npx shadcn@latest add <component-name>
```

Examples:

```bash
npx shadcn@latest add dialog
npx shadcn@latest add dropdown-menu
npx shadcn@latest add table
```

The CLI reads `components.json` and installs or updates files under `shared/components/ui/` (and related paths per the schema).

## What gets configured

`components.json` in this repo sets:

- **Style:** `new-york`
- **RSC:** `true`
- **Tailwind:** `shared/styles/globals.css` as the CSS entry
- **Aliases:** `components` → `@/shared/components`, `ui` → `@/shared/components/ui`, `utils` → `@/shared/lib/utils`, `lib` → `@/shared/lib`, `hooks` → `@/shared/hooks`
- **Icons:** Lucide (`iconLibrary`: `lucide`)

## After adding

1. Import from `@/shared/components/ui/<name>`.
2. If the new component depends on `cn`, it should use `@/shared/lib/utils` (the CLI respects the alias).
3. Run the dev server or build to confirm there are no duplicate or conflicting exports.

## Updating existing components

Re-running `add` for a component name that already exists may overwrite local changes. Commit or diff before upgrading. For upstream breaking changes, follow the [shadcn changelog](https://ui.shadcn.com/docs/changelog) and adjust imports if paths change.
