## Overview

Form UI lives under `shared/components/ui/`. Use low-level controls like `Input`, `Textarea`, `Select`, `Checkbox`, `Switch`, `RadioGroup`, and `NativeSelect`. For field layout, labels, descriptions, and errors, use the `Field*` primitives from `field.tsx`.

> **Note:** `Select`, `Checkbox`, `Switch`, `RadioGroup`, `Field`, and `InputGroup` parts are client components.

## Basic inputs

```tsx
import { Input } from "@/shared/components/ui/input";

<Input type="email" name="email" placeholder="you@example.com" />;
```

```tsx
import { Textarea } from "@/shared/components/ui/textarea";

<Textarea name="bio" rows={4} placeholder="Tell us about yourself" />;
```

```tsx
import { Label } from "@/shared/components/ui/label";

<Label htmlFor="email">Email</Label>;
```

## Select

```tsx
"use client";

import {
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
} from "@/shared/components/ui/select";

<Select defaultValue="one">
  <SelectTrigger className="w-[180px]">
    <SelectValue placeholder="Choose" />
  </SelectTrigger>
  <SelectContent>
    <SelectItem value="one">One</SelectItem>
    <SelectItem value="two">Two</SelectItem>
  </SelectContent>
</Select>;
```

## Field primitives

Use `Field`, `FieldLabel`, `FieldDescription`, and `FieldError` for form layout without adding another abstraction layer:

```tsx
<Field data-invalid={emailField.invalid}>
  <FieldLabel htmlFor="email">Email</FieldLabel>
  <Input
    id="email"
    name="email"
    defaultValue={emailField.value}
    aria-invalid={emailField.invalid}
    required
  />
  <FieldError>{emailField.error}</FieldError>
</Field>
```

`Field` also supports grouped layouts like `FieldGroup`, `FieldSet`, and `FieldLegend`.
