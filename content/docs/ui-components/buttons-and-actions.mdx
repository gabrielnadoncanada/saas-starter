## Import

```tsx
import { Button, buttonVariants } from "@/shared/components/ui/button";
```

`Button` is a styled `<button>` (or slotted child when `asChild` is true). It uses `class-variance-authority` for `variant` and `size`.

## Variants

| Variant       | Use case                                      |
| ------------- | --------------------------------------------- |
| `default`     | Primary actions (filled primary color)        |
| `destructive` | Delete, irreversible, or dangerous actions    |
| `outline`     | Secondary actions with border                   |
| `secondary`   | Lower-emphasis filled button                  |
| `ghost`       | Minimal chrome; toolbars, icon-adjacent actions |
| `link`        | Text link appearance                          |

Default is `variant="default"`.

```tsx
import { Button } from "@/shared/components/ui/button";

<div className="flex flex-wrap gap-2">
  <Button>Default</Button>
  <Button variant="secondary">Secondary</Button>
  <Button variant="outline">Outline</Button>
  <Button variant="ghost">Ghost</Button>
  <Button variant="destructive">Destructive</Button>
  <Button variant="link">Link</Button>
</div>
```

## Sizes

| Size       | Typical use                          |
| ---------- | ------------------------------------ |
| `default`  | Standard height (`h-9`)              |
| `xs`       | Compact rows, dense UI             |
| `sm`       | Slightly smaller than default        |
| `lg`       | Prominent CTAs                       |
| `icon`     | Square icon button (`size-9`)        |
| `icon-xs`  | Small square icon                    |
| `icon-sm`  | Medium-small icon                    |
| `icon-lg`  | Large icon                           |

Default is `size="default"`.

```tsx
<Button size="sm">Small</Button>
<Button size="icon" aria-label="Close">
  <XIcon />
</Button>
```

## Polymorphic usage (`asChild`)

Render as another element (for example `Link`) while keeping button styles:

```tsx
import Link from "next/link";
import { Button } from "@/shared/components/ui/button";

<Button asChild>
  <Link href="/dashboard">Go to dashboard</Link>
</Button>
```

## `buttonVariants` for non-button elements

Use when you need the same look on a different tag or component:

```tsx
import { buttonVariants } from "@/shared/components/ui/button";
import { cn } from "@/shared/lib/utils";

<a className={cn(buttonVariants({ variant: "outline", size: "sm" }))} href="#">
  Styled as button
</a>
```

## Accessibility

Invalid form fields can set `aria-invalid` on controls; the button styles include `aria-invalid` variants for destructive focus rings where relevant. Icon-only buttons should include `aria-label` (or visible text).
