## App shell (`AppShell`)

`shared/components/layout/shell/app-shell.tsx` composes the authenticated dashboard chrome:

- `SearchProvider` (command palette context)
- `TooltipProvider`
- `SidebarProvider` with `defaultOpen`
- `SkipToMain` (a11y skip link)
- Your **sidebar** as `React.ReactNode`
- `SidebarInset` wrapping the main column
- `Header` with the global **Search** trigger inside

```tsx
import { AppShell } from "@/shared/components/layout/shell/app-shell";
import { YourSidebar } from "./your-sidebar";

export function Layout({ children }: { children: React.ReactNode }) {
  return (
    <AppShell defaultOpen sidebar={<YourSidebar />}>{children}</AppShell>
  );
}
```

> **Note:** `AppShell` is a client component. Pass sidebar content that is already valid as client or server children per Next.js rules.

## Header (`Header`)

`shared/components/layout/shell/header.tsx` renders a slim top bar: `SidebarTrigger` (outline) plus any children (the starter passes `Search`).

```tsx
import { Header } from "@/shared/components/layout/shell/header";

<Header>
  <span className="text-sm text-muted-foreground">Optional slot</span>
</Header>
```

## Page layout (`Page` and helpers)

`shared/components/layout/page-layout.tsx` exports layout primitives for content inside the shell:

- **`Page`** — `div` with padding and optional `container` width. Props: `fixed` (full-height flex column with overflow control), `fluid` (skip `container` when true).
- **`PageHeader`** — grid header row; pairs with `PageTitle`, `PageDescription`, `PageHeaderActions`.
- **`PageTitle`** — large heading style.
- **`PageDescription`** — muted subtext.

```tsx
import {
  Page,
  PageDescription,
  PageHeader,
  PageHeaderActions,
  PageTitle,
} from "@/shared/components/layout/page-layout";
import { Button } from "@/shared/components/ui/button";

<Page>
  <PageHeader>
    <PageTitle>Tasks</PageTitle>
    <PageDescription>Manage your tasks.</PageDescription>
    <PageHeaderActions>
      <Button size="sm">New task</Button>
    </PageHeaderActions>
  </PageHeader>
  {/* page body */}
</Page>
```

Use `fixed` on `Page` when the route needs a constrained scroll region (e.g. full-viewport panels).

## Related: sidebar UI

The collapsible app sidebar (groups, menu buttons, mobile sheet) is implemented in `shared/components/ui/sidebar.tsx` and is what `AppShell`/`SidebarProvider` expect alongside your navigation implementation.
