## Types (`sidebar-types.ts`)

`shared/components/navigation/sidebar-types.ts` defines data shapes for sidebar config:

- **`SidebarNavLink`** — `title`, `url`, optional `icon`, `badge`; no nested `items`.
- **`SidebarNavCollapsible`** — `title`, optional `icon`, `badge`, and `items` (each child has `url`).
- **`SidebarNavGroup`** — `title` + `items: SidebarNavItem[]`.
- **`SidebarData`** — `{ navGroups: SidebarNavGroup[] }`.

Use these types when building `Sidebar` content from configuration.

## Nav group (`NavGroup`)

`NavGroup` renders one labeled section of links using `shared/components/ui/sidebar` primitives. It:

- Highlights active routes via `usePathname()`.
- Renders flat links with `SidebarMenuButton` + Next.js `Link`.
- Renders nested items as a **collapsible** when the sidebar is expanded, or as a **dropdown** when the desktop sidebar is **collapsed** (icon mode).

```tsx
"use client";

import { LayoutDashboard, Settings } from "lucide-react";
import { NavGroup } from "@/shared/components/navigation/nav-group";

const group = {
  title: "Main",
  items: [
    { title: "Dashboard", url: "/dashboard", icon: LayoutDashboard },
    {
      title: "Settings",
      icon: Settings,
      items: [
        { title: "Profile", url: "/settings/profile" },
        { title: "Billing", url: "/settings/billing" },
      ],
    },
  ],
};

<NavGroup {...group} />;
```

> **Note:** `NavGroup` is a client component because it uses pathname and sidebar context.

## Settings sidebar (`SettingsSidebar`)

`SettingsSidebar` is a concrete sidebar for the settings area: floating variant, icon collapsible, “Back to dashboard” in the header, and nav groups driven by `routes` + `accountFlags` (team features toggle workspace links).

```tsx
import { SettingsSidebar } from "@/shared/components/navigation/settings-sidebar";

// Typically passed as the `sidebar` prop to AppShell on settings layouts
<SettingsSidebar />
```

Customize by editing `settingsSidebarData` in that file or by introducing your own sidebar that reuses `NavGroup`.

## Search (`Search`)

`Search` is a header button that opens the command palette via `useSearch()` from `@/shared/components/command/search-provider`. It shows a ⌘K hint on larger breakpoints.

```tsx
import { Search } from "@/shared/components/navigation/search";

<Search placeholder="Search…" />
```

`AppShell` already mounts `Search` inside `Header`. Use this component elsewhere only if you also wrap the tree with `SearchProvider` (as `AppShell` does).
