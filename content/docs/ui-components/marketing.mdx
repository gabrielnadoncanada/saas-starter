## Overview

Marketing pages compose sections from `features/marketing/components/`. These are **not** generic shadcn exports—they are tailored to the starter’s landing content and styling (Tailwind, `Button` from `shared/components/ui`, `routes` from `shared/constants/routes`).

> **Note:** Replace branding (logo block, product name copy) inside these files when you ship your own product.

## Header and footer

- **`MarketingHeader`** — Sticky header with logo, anchor links to on-page sections (`/#features`, `/#pricing`, etc.), desktop CTAs (Demo → sign-in, “Get the Starter” → pricing), and a mobile menu toggle.
- **`MarketingFooter`** — Multi-column footer: brand blurb, Product / Resources / Legal link lists.

```tsx
import { MarketingFooter } from "@/features/marketing/components/marketing-footer";
import { MarketingHeader } from "@/features/marketing/components/marketing-header";

<>
  <MarketingHeader />
  <main>{/* sections */}</main>
  <MarketingFooter />
</>
```

## Feature and pricing

- **`MarketingFeatureGrid`** — Grid of feature cards with Lucide icons and short descriptions (plan gating, Stripe, auth, teams, dashboard, Prisma, settings, AI assistant).
- **`StarterPricing`** — Three pricing tiers (`Starter`, `Pro`, `Agency`) with feature bullets, highlighted tier, and CTA buttons; data is defined inline in the component.

```tsx
import { MarketingFeatureGrid } from "@/features/marketing/components/marketing-feature-grid";
import { StarterPricing } from "@/features/marketing/components/starter-pricing";

<section id="features">
  <MarketingFeatureGrid />
</section>
<section id="pricing">
  <StarterPricing />
</section>
```

## Content sections

- **`ComparisonSection`** — Feature comparison table (this starter vs alternatives); uses icons for yes/no/partial cells.
- **`BuyerFaq`** — Expandable FAQ list with local state (not the shadcn `Accordion` component).
- **`CodeProof`** — Two-column section explaining plan gating APIs with a code-like list.
- **`AiProof`** — Similar pattern for the AI assistant value proposition.
- **`IncludedItemsSection`**, **`BuilderSection`**, **`ScreenshotsGallery`** — Additional landing sections (lists, builder narrative, image gallery).

Import paths follow `@/features/marketing/components/<file-name>` (e.g. `buyer-faq` exports `BuyerFaq`).

## Customization

1. Edit copy and arrays in each component file to match your offer.
2. Point CTAs to your checkout or signup routes (`routes` constants) where applicable.
3. Keep using shared UI primitives (`Button`, etc.) for consistent styling with the authenticated app.
