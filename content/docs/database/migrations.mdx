---
title: Migrations and database scripts
description: pnpm scripts for Prisma migrate, push, generate, seed, and related workflows.
---

Database workflows are driven by **`package.json`** scripts that wrap the Prisma CLI.

## Core commands

```json
"db:setup": "pnpm exec tsx shared/lib/db/setup.ts",
"db:format": "pnpm exec prisma format",
"db:generate": "pnpm exec prisma generate",
"db:migrate": "pnpm exec prisma migrate dev",
"db:migrate:create": "pnpm exec prisma migrate dev --create-only",
"db:push": "pnpm exec prisma db push",
"db:seed": "pnpm exec prisma db seed",
"db:reset": "pnpm exec prisma migrate reset",
"db:studio": "pnpm exec prisma studio"
```

### `pnpm db:migrate`

Runs **`prisma migrate dev`**: applies pending migrations in development, prompts for a name when there are schema changes, and regenerates the client as needed. Use this during normal feature work when you want versioned SQL in `prisma/migrations/`.

### `pnpm db:migrate:create`

Runs **`prisma migrate dev --create-only`**: creates a new migration file without applying it—useful when you want to edit SQL before applying.

### `pnpm db:push`

Runs **`prisma db push`**: pushes the current schema to the database **without** creating migration files. Handy for rapid local iteration; for anything shared or production-like, prefer migrations.

### `pnpm db:generate`

Runs **`prisma generate`**: regenerates `@prisma/client` from the current schema. Also runs automatically on **`pnpm install`** via the `postinstall` script.

## Seed and reset

### `pnpm db:seed`

Runs the seed command configured in `prisma.config.ts` (`tsx shared/lib/db/seed.ts`). Use after migrations when you need demo data, admin workspace, or Stripe product seeds.

### `pnpm db:reset`

Runs **`prisma migrate reset`**: drops the database, reapplies all migrations, and runs the seed. Destructive—intended for local development.

## Other scripts

| Script | Purpose |
| ------ | ------- |
| `pnpm db:format` | Format Prisma schema files |
| `pnpm db:studio` | Open Prisma Studio GUI |
| `pnpm db:setup` | Interactive `.env` / Docker Postgres / Stripe helper (see `shared/lib/db/setup.ts`) |

> **Warning:** `db:reset` deletes local data. Do not use it against production databases.

> **Note:** First-time local setup often runs `pnpm db:migrate` then `pnpm db:seed` after configuring `POSTGRES_URL` (for example via `pnpm db:setup`).
