## Singleton client (`shared/lib/db/prisma.ts`)

The app uses a **single shared `PrismaClient`** instance to avoid exhausting connections in development (hot reload) and to configure logging consistently.

```ts
import { PrismaPg } from "@prisma/adapter-pg";
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

const adapter = new PrismaPg({
  connectionString: process.env.POSTGRES_URL,
});

export const db =
  globalForPrisma.prisma ??
  new PrismaClient({
    adapter,
    log: process.env.NODE_ENV === "development" ? ["error", "warn"] : ["error"],
  });

if (process.env.NODE_ENV !== "production") {
  globalForPrisma.prisma = db;
}
```

**Patterns**

- Import `db` from `@/shared/lib/db/prisma` (or the project’s alias) in server code, server actions, and route handlers—not per-request `new PrismaClient()`.
- **`POSTGRES_URL`** must be set; the client uses the **`@prisma/adapter-pg`** adapter with the `pg` driver.

> **Warning:** Only use `db` in server-side code. Do not import it into client components.

## Enum re-exports (`shared/lib/db/enums.ts`)

Feature code should import Prisma enums and org-role helpers from here instead of deep-importing `@prisma/client` internals. The file re-exports generated enums such as `PlatformRole`, `TaskLabel`, `TaskPriority`, `TaskStatus`, and defines **`ORG_ROLES`** / **`OrgRole`** helpers for organization member roles (stored as strings in the DB).

## Seed (`shared/lib/db/seed.ts`)

`pnpm db:seed` runs this script. It orchestrates modular seeds:

- `seeds/admin-seed.ts`
- `seeds/demo-workspace-seed.ts`
- `seeds/stripe-products-seed.ts`

(Additional content seeds may exist under `shared/lib/db/seeds/`.)

## Interactive setup (`shared/lib/db/setup.ts`)

`pnpm db:setup` runs a guided script that can help configure **Postgres** (including Docker via `docker-compose.yml`), **Stripe** keys and webhook secret, **`AUTH_SECRET`**, and write **`.env`** (or `.env.generated` if `.env` already exists). It is not required at runtime—only for bootstrapping a machine.
