---
title: Database schema
description: Overview of each Prisma model file and the models they define.
---

The database is PostgreSQL. Models are grouped into files under `prisma/models/`. Cross-file relations (for example `User` → `Member` → `Organization`) are valid because Prisma merges all files.

## `auth.prisma`

**Enums**

- `PlatformRole` — `USER`, `SUPER_ADMIN`

**Models**

- **`User`** — Core user: email, profile (`name`, `image`), `emailVerified`, `platformRole`, string `role`, locale, 2FA flag, ban fields, Stripe customer id, soft delete (`deletedAt`), optional phone.
- **`Account`** — OAuth/credential provider link: `providerId`, `accountId`, tokens, optional `password` hash; relates to `User`.
- **`Session`** — Server session: `token`, `expiresAt`, optional `activeOrganizationId`, `impersonatedBy`; relates to `User`.
- **`Verification`** — Email and other verification tokens (`identifier` + `value`, `expiresAt`).

## `teams.prisma`

**Models**

- **`Organization`** — Tenant: `id`, `name`, optional `slug`, `logo`, `metadata`, `stripeCustomerId`; relates to members, invitations, tasks, usage, purchases, AI conversations, files.
- **`Member`** — Membership: `userId`, `organizationId`, string `role` (Better Auth organization plugin); unique per org+user.
- **`Invitation`** — Pending invites: email, role, status, `inviterId`, `expiresAt`.

## `billing.prisma`

**Models**

- **`Subscription`** — Stripe-backed subscription: `plan`, `referenceId`, Stripe ids, status, period/trial/cancel fields, optional `seats`, `items` relation.
- **`SubscriptionItem`** — Line items: `itemType`, `itemKey`, `componentKey`, Stripe price/item ids, quantity, status; unique per subscription + item dimensions.
- **`Purchase`** — One-off purchases tied to `organizationId`: `purchaseType`, `itemKey`, Stripe checkout/payment ids, amount, currency, status.

## `tasks.prisma`

**Enums**

- `TaskStatus`, `TaskLabel`, `TaskPriority`

**Models**

- **`Task`** — Example domain model per organization: human-readable `code`, title, description, status/label/priority, indexes for listing.

## `ai.prisma`

**Models**

- **`AiConversation`** — Assistant threads: `organizationId`, `createdByUserId`, `surface`, `title`, `messagesJson` (`Json`), `lastMessageAt`.

## `platform.prisma`

**Models**

- **`StoredFile`** — Uploaded file metadata: `storageKey`, `fileName`, `contentType`, `sizeBytes`; scoped to org and uploader.
- **`TwoFactor`** — TOTP/2FA secret and `backupCodes` for a `User`.

## `usage.prisma`

**Models**

- **`UsageCounter`** — Metered usage per org: `limitKey`, `periodStart`, `count`; unique per org + key + period start.

> **Note:** Organization member roles in `Member.role` are plain strings (`owner`, `admin`, `member`); app-level helpers for these live in `shared/lib/db/enums.ts` alongside Prisma enum re-exports.
