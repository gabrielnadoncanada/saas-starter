## Email and password

Configured in `shared/lib/auth/auth-config.ts`:

```ts title="shared/lib/auth/auth-config.ts"
emailAndPassword: {
  enabled: true,
  requireEmailVerification: true,
  minPasswordLength: 8,
  sendResetPassword: async ({ user, url }) => {
    // send email
  },
},
```

The UI no longer uses a client-side auth wrapper. Public auth forms call server actions in `features/auth/actions/public-auth.actions.ts`, and those actions call Better Auth server APIs directly:

- `auth.api.signInEmail`
- `auth.api.signUpEmail`
- `auth.api.requestPasswordReset`
- `auth.api.resetPassword`
- `auth.api.sendVerificationEmail`

Signed-in password changes still use `savePasswordAction`, which calls `auth.api.changePassword` or `auth.api.setPassword`.

## Magic link

The `magicLink` plugin is registered in `auth-config.ts`. The sign-in and sign-up pages call `authClient.signIn.magicLink` from the client for the secondary "email link" button, then redirect to `/check-email`.

## OAuth

OAuth buttons call `authClient.signIn.social` directly from the sign-in and sign-up forms. Provider enablement and labels stay in `shared/lib/auth/oauth-config.ts`.
