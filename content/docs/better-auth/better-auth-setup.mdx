## Entry points

- **Server:** `shared/lib/auth/auth-config.ts` exports `auth` from `betterAuth(...)`.
- **Re-export:** Root `auth.ts` re-exports for libraries that import from `@/auth` or the package root pattern.
- **API route:** `app/api/auth/[...all]/route.ts` wires the handler:

```1:6:app/api/auth/[...all]/route.ts
import { toNextJsHandler } from "better-auth/next-js";

import { auth } from "@/shared/lib/auth/auth-config";

export const { GET, POST } = toNextJsHandler(auth);
```

- **Client:** `shared/lib/auth/auth-client.ts` uses `createAuthClient` from `better-auth/react` with plugins that mirror the server.

## Core options

```18:24:shared/lib/auth/auth-config.ts
export const auth = betterAuth({
  database: prismaAdapter(db, {
    provider: "postgresql",
  }),
  secret: process.env.AUTH_SECRET,
  baseURL: process.env.BASE_URL,
  basePath: "/api/auth",
```

Sessions last **24 hours** (`expiresIn: 60 * 60 * 24`) with **`updateAge` 1 hour** ‚Äî see `session` in `auth-config.ts`.

## User fields

Additional user fields include `role`, `phoneNumber`, and `twoFactorEnabled` (all optional, returned on the user object) for app RBAC and security UI.

## Email flows

Email/password and verification emails are configured as follows:

```45:72:shared/lib/auth/auth-config.ts
  emailAndPassword: {
    enabled: true,
    requireEmailVerification: true,
    minPasswordLength: 8,
    sendResetPassword: async ({ user, url }) => {
      await sendEmail({
        to: [user.email],
        subject: "Reset your password",
        react: createElement(ResetPasswordTemplate, { resetUrl: url }),
        text: `Reset your password: ${url}`,
        tags: [{ name: "email_type", value: "reset_password" }],
      });
    },
  },

  emailVerification: {
    sendOnSignUp: true,
    autoSignInAfterVerification: true,
    sendVerificationEmail: async ({ user, url }) => {
      await sendEmail({
        to: [user.email],
        subject: "Verify your email address",
        react: createElement(VerifyEmailTemplate, { verificationUrl: url }),
        text: `Verify your email address: ${url}`,
        tags: [{ name: "email_type", value: "verify_email" }],
      });
    },
  },
```

Plugins add **admin**, **magic link**, **organization**, **twoFactor**, and **`nextCookies()`** ‚Äî see the other pages in this section.

> **Note:** Email sending uses `shared/lib/email/client` and React email templates (`VerifyEmailTemplate`, `ResetPasswordTemplate`, `MagicLinkEmail`, etc.).
