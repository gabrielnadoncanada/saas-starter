## Server configuration

The `admin` plugin sets the default user role and which roles count as **administrators** for Better Auth’s admin APIs:

```106:109:shared/lib/auth/auth-config.ts
    admin({
      defaultRole: "user",
      adminRoles: [...PLATFORM_ADMIN_ROLES],
    }),
```

`PLATFORM_ADMIN_ROLES` is defined in `shared/lib/auth/roles.ts`:

```30:34:shared/lib/auth/roles.ts
export const PLATFORM_ADMIN_ROLES = ["admin"] as const;

export function isPlatformAdmin(value: unknown): boolean {
  return hasAnyRole(value, PLATFORM_ADMIN_ROLES);
}
```

`parseRoles` supports comma-separated role strings when reading `role` from the session.

## Client

`adminClient()` is registered in `shared/lib/auth/auth-client.ts` so typed admin methods are available where needed.

## Access control in the app

Routes under `/admin` use `requireAdmin()` / `requireAdminAction()` from `features/auth/server/require-admin.ts`, which checks `isPlatformAdmin(user.role)` and redirects or throws if the user is not a platform admin.

## Changing roles and moderation

Admin server actions call **`auth.api`** methods from the admin plugin, for example:

```77:90:features/admin/actions/admin-users.actions.ts
export async function setUserRoleAction(
  userId: string,
  role: "user" | "admin",
) {
  const adminId = await requireAdminAction();

  if (userId === adminId) {
    throw new Error("You cannot change your own role");
  }

  await auth.api.setRole({
    body: { userId, role },
    headers: await headers(),
  });

  revalidateAdminUsersPage();
}
```

Other actions in the same file use `banUser`, `unbanUser`, `removeUser`, `revokeUserSession`, etc. — all gated by `requireAdminAction()`.

## Bootstrap admin user

The seed script `shared/lib/db/seeds/admin-seed.ts` creates or updates a user with **`role: "admin"`** for local development. Use that pattern or the admin UI (once you have one admin) to grant the `admin` role — it must match `PLATFORM_ADMIN_ROLES`.

> **Note:** Organization membership roles (`owner`, `member`, etc.) are separate from the platform **`user` / `admin`** role on the Better Auth user record.
