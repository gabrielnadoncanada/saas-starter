This project uses **Tailwind CSS v4** with the PostCSS plugin. There is no `tailwind.config.js`; configuration lives in CSS files under `shared/styles/`.

## Entry point

`app/layout.tsx` imports a single global stylesheet:

```1:1:app/layout.tsx
import "@/shared/styles/globals.css";
```

`shared/styles/globals.css` imports Tailwind, layers base styles, and pulls in theme and component CSS:

```9:21:shared/styles/globals.css
/* Tailwind CSS */
@import "tailwindcss";

/* local styles - update the below if you add a new style */
@import "./theme.css";
@import "./theme.utilities.css";
@import "./shadcn-ui.css";
@import 'fumadocs-ui/css/neutral.css';
@import 'fumadocs-ui/css/preset.css';
/* plugins - update the below if you add a new plugin */

/* variants - update the below if you add a new variant */
@variant dark (&:where(.dark, .dark *));
```

PostCSS is configured with `@tailwindcss/postcss` in `postcss.config.mjs`.

## Semantic colors (CSS variables)

UI colors are defined as custom properties on `:root` and `.dark` in `shared/styles/shadcn-ui.css`. They use **oklch** values and map to shadcn conventions (`--background`, `--primary`, `--sidebar-*`, charts, etc.).

```9:22:shared/styles/shadcn-ui.css
@layer base {
  :root {
    --font-sans: -apple-system, BlinkMacSystemFont, var(--font-sans-fallback);
    --font-heading: var(--font-sans);

    --background: oklch(100% 0 0);
    --foreground: oklch(14.5% 0 0);
    --card: oklch(100% 0 0);
    --card-foreground: oklch(14.5% 0 0);
    --popover: oklch(100% 0 0);
    --popover-foreground: oklch(14.5% 0 0);
    --primary: oklch(20.5% 0 0);
    --primary-foreground: oklch(98.5% 0 0);
```

> **Note:** The file header points to [shadcn themes](https://ui.shadcn.com/themes). When copying palette values from the theme generator, apply them in the **hsl(...)** form expected by that workflow, or keep **oklch** and adjust consistently.

## Tailwind `@theme` bridge

`shared/styles/theme.css` maps those CSS variables into Tailwind theme tokens (e.g. `bg-background`, `text-muted-foreground`, `rounded-lg` via `--radius-*`). Edit both `shadcn-ui.css` (raw tokens) and `theme.css` (if you add new mapped keys) when extending the design system.

## Utilities and layout

`shared/styles/theme.utilities.css` defines a `container` utility used for page width and horizontal padding.

## Dark mode

Dark styles apply when an ancestor has the `dark` class (see the `@variant dark` rule in `globals.css`). The `Providers` tree typically sets theme mode; components use `bg-background`, `text-foreground`, and related tokens so light and dark stay in sync.
