## Root layout: `next/font/google`

The app uses **Inter** from `next/font/google`. The font instance exposes a CSS variable `--font-sans-fallback` and is attached to `<html>` via `className`.

```24:40:app/layout.tsx
const inter = Inter({
  subsets: ["latin"],
  variable: "--font-sans-fallback",
  fallback: ["system-ui,Helvetica Neue,Helvetica,Arial"],
});

export default async function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang={"en"} suppressHydrationWarning className={inter.variable}>
      <body suppressHydrationWarning>
        <Providers>{children}</Providers>
      </body>
    </html>
  );
}
```

Next.js inlines optimized `@font-face` rules for Inter and sets `--font-sans-fallback` on the element that receives `inter.variable` (here, the root `<html>`).

## Global CSS: sans and heading stacks

`shared/styles/shadcn-ui.css` wires the loaded variable into the stack used across the UI:

```10:12:shared/styles/shadcn-ui.css
  :root {
    --font-sans: -apple-system, BlinkMacSystemFont, var(--font-sans-fallback);
    --font-heading: var(--font-sans);
```

`shared/styles/theme.css` exposes `--font-sans` and `--font-heading` to Tailwind’s theme so utility classes resolve correctly.

## Changing the font

1. In `app/layout.tsx`, import a different font from `next/font/google` (or `next/font/local` for self-hosted files).
2. Keep or rename the `variable` string. If you rename it (e.g. `--font-display`), update `--font-sans-fallback` references in `shadcn-ui.css` to match, **or** set `--font-sans` to use your new variable directly.
3. Adjust `subsets` and `fallback` arrays for your languages and metrics.

Example shape (illustrative only):

```tsx
import { DM_Sans } from "next/font/google";

const dmSans = DM_Sans({
  subsets: ["latin"],
  variable: "--font-sans-fallback",
  fallback: ["system-ui", "sans-serif"],
});
```

Then use `className={dmSans.variable}` on `<html>` as today.

> **Note:** Fumadocs documentation pages import their own CSS presets in `globals.css`; body text still inherits from your root font stack unless Fumadocs overrides specific elements.
