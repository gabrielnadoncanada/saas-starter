## Server data

The dashboard page renders `DashboardOverview` (`features/dashboard/components/dashboard-overview.tsx`), which loads data via `getDashboardOverview()` (`features/dashboard/server/get-dashboard-overview.ts`).

That function resolves the **current organization** with `getCurrentOrganization()` and loads entitlements, then queries Prisma **filtered by `organizationId`** for:

- Task count and five most recent tasks
- Monthly task usage (`getMonthlyUsage`)
- AI conversation count
- Tasks created in the last seven days (for the activity chart)

```ts
const organizationId = organization?.id ?? null;

const [taskCount, recentTasks, tasksUsage, assistantConversationCount, recentTaskHistory] =
  organizationId
    ? await Promise.all([
        db.task.count({ where: { organizationId } }),
        db.task.findMany({ where: { organizationId }, orderBy: { createdAt: "desc" }, take: 5 }),
        // ...
      ])
    : [0, [], 0, 0, []];
```

## What the UI shows

For the active workspace, the overview includes:

- **Current plan** card (from catalog + org billing interval when present)
- **Task creations this month** with a usage meter against plan limits
- **Organization members** count
- **AI assistant** card when the plan includes `ai.assistant`
- **Task activity this week** (bar chart from recent task `createdAt` values)
- **Getting started** checklist (first task, invite teammate, use assistant—items hidden when not applicable)
- **Recent tasks** list
- Optional **upgrade** prompt when `team.analytics` is not available
- Links to Tasks, AI Assistant, and **Organization Settings**

The page title uses the organization name in the welcome line when available.

> **Note:** If there is no active organization context, counts fall back to zeros and the checklist still renders where data allows.
