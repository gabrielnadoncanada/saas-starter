## Data model

Organizations are first-class tenants. Core tables in `prisma/models/teams.prisma`:

```prisma
model Organization {
  id        String   @id
  name      String   @db.VarChar(100)
  slug      String?  @unique
  logo      String?
  metadata  String?  @db.Text
  createdAt DateTime @default(now())

  stripeCustomerId String? @unique

  members             Member[]
  invitations         Invitation[]
  tasks               Task[]
  usageCounters       UsageCounter[]
  purchases           Purchase[]
  aiConversations     AiConversation[]
  storedFiles         StoredFile[]
}

model Member {
  id             String   @id
  organizationId String
  userId         String
  role           String
  createdAt      DateTime @default(now())

  organization Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  user         User         @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([organizationId, userId])
  @@index([userId])
  @@index([organizationId])
}

model Invitation {
  id             String   @id
  organizationId String
  email          String   @db.VarChar(255)
  role           String?
  status         String
  inviterId      String
  expiresAt      DateTime
  createdAt      DateTime @default(now())

  organization Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  inviter      User         @relation("InvitationSender", fields: [inviterId], references: [id], onDelete: Cascade)

  @@index([organizationId])
  @@index([email])
}
```

## Session and “active” organization

The signed-in session stores which workspace is active (`activeOrganizationId`). Server code loads org-scoped data for that id—for example `getCurrentOrganization()` in `features/organizations/server/current-organization.ts`, which uses Better Auth’s `getFullOrganization` plus billing snapshot.

## Account mode (UI only)

`shared/config/account.config.ts` toggles whether the product feels B2C (“personal-only”) or B2B (“organizations-only”). Regardless of mode, **data stays organization-scoped**; the mode only changes visible features (switcher, team pages, creating extra orgs):

```ts
export const accountFlags = {
  enableTeamFeatures: accountMode !== "personal-only",
  allowCreateOrganization: accountMode !== "personal-only",
  showOrgSwitcher: accountMode !== "personal-only",
  // ...
};
```

> **Note:** Set `NEXT_PUBLIC_ACCOUNT_MODE` to `personal-only` or `organizations-only` (default is `organizations-only`).

## Multi-tenant behavior

- **Tasks, usage, AI conversations, files, and purchases** are tied to an `organizationId` in the database (see relations on `Organization` in `teams.prisma`).
- Switching the active organization changes which rows apply for the current user session after navigation/refresh.
