## Standard pattern

Use one form pattern across the app:

- Zod schema in the feature
- server action wrapped with `validatedAuthenticatedAction` or `validatedPublicAction`
- client form component using `useActionState`
- `FormActionState` for `success`, `error`, `values`, and `fieldErrors`
- `getFieldState()` for field-level UI

The tasks feature shows the baseline:

```tsx title="features/tasks/components/task-form.tsx"
const createState = useActionState<CreateTaskActionState, FormData>(
  createTaskAction,
  {},
);
const updateState = useActionState<FormActionState<UpdateTaskValues>, FormData>(
  updateTaskAction,
  {},
);
const [state, formAction, isPending] =
  mode === "create" ? createState : updateState;
```

```tsx title="features/tasks/components/task-form.tsx"
<form action={formAction}>
  <Field data-invalid={title.invalid}>
    <FieldLabel htmlFor="task-title">Title</FieldLabel>
    <Input
      id="task-title"
      name="title"
      defaultValue={title.value}
      aria-invalid={title.invalid}
      required
    />
    <FieldError>{title.error}</FieldError>
  </Field>
</form>
```

## Public vs authenticated actions

- Use `validatedAuthenticatedAction` for signed-in mutations.
- Use `validatedPublicAction` for auth flows like sign-in, sign-up, forgot password, and reset password.

Authenticated example:

```ts title="features/tasks/actions/task.actions.ts"
export const updateTaskAction = validatedAuthenticatedAction(
  updateTaskSchema,
  async (data, _formData, user) => {
    return updateTask(data, user.id);
  },
);
```

Public example:

```ts title="features/auth/actions/public-auth.actions.ts"
export const signInAction = validatedPublicAction(
  signInFormSchema,
  async ({ email, password, callbackUrl }) => {
    await auth.api.signInEmail({
      headers: await headers(),
      body: {
        email: email.trim().toLowerCase(),
        password,
        callbackURL: callbackUrl,
      },
    });

    return { redirectTo: callbackUrl ?? routes.auth.postSignIn };
  },
);
```

## Shared state shape

Server actions return `FormActionState` values (`success`, `error`, `fieldErrors`, optional `values`) defined in `shared/types/form-action-state.ts`.

Auth pages, CRUD dialogs, and settings forms now all follow this same contract.
