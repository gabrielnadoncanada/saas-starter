## Prisma client

The shared client is `db` from `shared/lib/db/prisma.ts`, configured with the PostgreSQL adapter and dev logging:

```ts title="shared/lib/db/prisma.ts"
import { PrismaPg } from "@prisma/adapter-pg";
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

const adapter = new PrismaPg({
  connectionString: process.env.POSTGRES_URL,
});

export const db =
  globalForPrisma.prisma ??
  new PrismaClient({
    adapter,
    log: process.env.NODE_ENV === "development" ? ["error", "warn"] : ["error"],
  });
```

Import `db` from feature `server/` files only; those files should start with `import "server-only"` when they must never run on the client (see `features/tasks/server/task-mutations.ts`).

## Queries in feature `server/` folders

Feature code reads and writes data through small functions. Example: listing tasks scopes by the current organization:

```ts title="features/tasks/server/task-mutations.ts"
export async function listTasks() {
  const organizationId = await getOrganizationId();

  return db.task.findMany({
    where: { organizationId },
    orderBy: { createdAt: "desc" },
  });
}
```

Admin org listing uses `db.organization.findMany` with `include` for members and `_count` (`features/organizations/server/admin-organizations.ts`).

## Transactions

Use `db.$transaction` when multiple steps must succeed or fail together. Task creation consumes usage and creates a row inside one transaction:

```ts title="features/tasks/server/task-mutations.ts"
  return db.$transaction(async (tx) => {
    await consumeMonthlyUsage(
      entitlements.organizationId,
      "tasksPerMonth",
      entitlements,
      { db: tx },
    );

    for (let attempt = 0; attempt < MAX_TASK_CODE_ATTEMPTS; attempt += 1) {
      const latestTask = await tx.task.findFirst({
        where: {
          organizationId: entitlements.organizationId,
        },
        orderBy: {
          id: "desc",
        },
        select: {
          code: true,
        },
      });
      ...
        return await tx.task.create({
          data: {
            organizationId: entitlements.organizationId,
            ...
          },
        });
```

Passing `{ db: tx }` into helpers lets billing code participate in the same transaction.

## Errors and safety

- Use `updateMany` / `deleteMany` with org scope in `where`, then check `count` to avoid touching other tenants’ rows (`task-mutations.ts`).
- Catch Prisma unique constraint errors (`P2002`) when retrying generated values (task code generation).

> **Note:** Schema files live under `prisma/models/`; run `pnpm db:migrate` or `pnpm db:push` after schema changes and regenerate the client with `pnpm db:generate`.
