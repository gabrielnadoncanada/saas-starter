## What account mode controls

The starter supports two **account modes**, selected with `NEXT_PUBLIC_ACCOUNT_MODE`:

- **`personal-only`** — B2C-style: one workspace per user, no team/invitation-focused UI.
- **`organizations-only`** — B2B-style: team features, org switcher, and organization creation (subject to auth plugin settings).

```ts title="shared/config/account.config.ts"
 * Controls how users interact with the platform:
 * - "personal-only"      → B2C: single workspace per user, no teams/invitations
 * - "organizations-only" → B2B: always team-based, org switcher visible
 *
 * Regardless of mode, data is always scoped to an organization under the hood.
 * The mode only controls what UI is shown to the user.
```

## Resolution and defaults

Valid values are enforced; anything else falls back to **`organizations-only`**.

```ts title="shared/config/account.config.ts"
function resolveAccountMode(): AccountMode {
  const raw = process.env.NEXT_PUBLIC_ACCOUNT_MODE;
  if (raw && VALID_MODES.includes(raw as AccountMode)) {
    return raw as AccountMode;
  }
  return "organizations-only";
}

export const accountMode = resolveAccountMode();
```

Set the variable at build time (it is `NEXT_PUBLIC_*` so it is available in the browser bundle).

## Feature flags (`accountFlags`)

All product behavior toggles derive from `accountMode`:

```ts title="shared/config/account.config.ts"
export const accountFlags = {
  /** Show team-related UI: members page, invitations, org settings. */
  enableTeamFeatures: accountMode !== "personal-only",

  /** Allow users to create additional organizations. */
  allowCreateOrganization: accountMode !== "personal-only",

  /** Show the workspace/org switcher in the sidebar. */
  showOrgSwitcher: accountMode !== "personal-only",

  /** Billing is attached to the organization (always true — org is the scope). */
  enableOrgBilling: true,

  /** The mode itself, for edge cases. */
  mode: accountMode,
} as const;
```

## Auth integration

The Better Auth `organization` plugin uses `allowCreateOrganization` from `accountFlags`:

```ts title="shared/lib/auth/auth-config.ts"
    organization({
      allowUserToCreateOrganization: accountFlags.allowCreateOrganization,
      creatorRole: "owner",
```

UI that should hide in personal-only mode (for example settings sidebar workspace links) gates on `accountFlags.enableTeamFeatures` — see `shared/components/navigation/settings-sidebar.tsx`.
