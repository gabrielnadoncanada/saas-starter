## Overview

Application-wide settings live in `shared/config/` as small, typed modules. Import them from feature code or server actions instead of scattering `process.env` reads.

> **Note:** `shared/config/billing.config.ts` is marked `server-only` and must not be imported from client components.

## Account configuration (`account.config.ts`)

`accountMode` and `accountFlags` control how the product behaves for end users (personal vs team UI). They are derived from `NEXT_PUBLIC_ACCOUNT_MODE`.

```ts title="shared/config/account.config.ts"
const VALID_MODES = ["personal-only", "organizations-only"] as const;
export type AccountMode = (typeof VALID_MODES)[number];

function resolveAccountMode(): AccountMode {
  const raw = process.env.NEXT_PUBLIC_ACCOUNT_MODE;
  if (raw && VALID_MODES.includes(raw as AccountMode)) {
    return raw as AccountMode;
  }
  return "organizations-only";
}

export const accountMode = resolveAccountMode();

/** Derived feature flags — all driven by the single account mode value. */
export const accountFlags = {
  /** Show team-related UI: members page, invitations, org settings. */
  enableTeamFeatures: accountMode !== "personal-only",

  /** Allow users to create additional organizations. */
  allowCreateOrganization: accountMode !== "personal-only",

  /** Show the workspace/org switcher in the sidebar. */
  showOrgSwitcher: accountMode !== "personal-only",

  /** Billing is attached to the organization (always true — org is the scope). */
  enableOrgBilling: true,

  /** The mode itself, for edge cases. */
  mode: accountMode,
} as const;
```

The Better Auth `organization` plugin reads `allowCreateOrganization` from these flags (see `shared/lib/auth/auth-config.ts`).

## Billing configuration (`billing.config.ts`)

`billingConfig` defines the **catalog** of plans and one-time products: capabilities, usage limits, marketing copy, and (when env price IDs exist) Stripe recurring schedules.

```ts title="shared/config/billing.config.ts"
export const billingConfig = {
  currency: "USD",
  plans: [
    {
      id: "free",
      name: "Free",
      description: "For trying the starter without paid billing.",
      pricingModel: "flat",
      features: ["Create tasks", "Billing portal access"],
      capabilities: ["task.create", "billing.portal"],
      limits: { tasksPerMonth: 10, teamMembers: 1, storageMb: 100 },
      schedules: {},
    },
```

Capabilities and limit keys are enumerated at the top of the file (`capabilities`, `limitKeys`). Paid plan Stripe price IDs come from environment variables such as `STRIPE_PRICE_PRO_MONTHLY`; if a price ID is missing, that interval’s schedule is omitted and the UI falls back accordingly.

> **Warning:** Changing plan `id` values or the shape of `billingConfig` affects database-stored subscription state and Stripe mapping. Coordinate with migrations and Stripe dashboard products when renaming plans.
