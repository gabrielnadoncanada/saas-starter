## Where it lives

- Page: `app/(auth)/sign-in/page.tsx`
- Form: `features/auth/components/sign-in-form.tsx`
- Public actions: `features/auth/actions/public-auth.actions.ts`

## Email and password

The sign-in form is a single `useActionState` form. It posts `email`, `password`, and `callbackUrl` to `signInAction`, which calls `auth.api.signInEmail`.

- Validation errors come back through `fieldErrors`.
- Invalid credentials render an inline form error.
- Unverified email returns `requiresVerification`, which shows `ResendVerificationForm`.
- Success returns `redirectTo`, and the client navigates to that URL.

## Magic link

When magic link is enabled, the secondary button calls `authClient.signIn.magicLink` and redirects the user to `/check-email` with the submitted email and preserved `callbackUrl`.

## OAuth

Enabled providers come from `getEnabledOAuthProviderIds()`. Clicking a provider calls `authClient.signIn.social`. Query param `?error=` is surfaced with `useToastMessage`.

## Related routes

| `/sign-in`     | Sign-in page          |
| `/check-email` | After magic link sent |
