title: Password reset
description: Request a reset link and set a new password with a token.
---

## Forgot password

- Page: `app/(auth)/forgot-password/page.tsx`
- Form: `features/auth/components/password/forgot-password-form.tsx`

The form uses `useActionState` with `requestPasswordResetAction`, which calls `auth.api.requestPasswordReset`.

- The submitted email is lowercased on the server.
- Success always shows the same neutral message so the UI does not leak whether the email exists.
- The action passes `routes.auth.resetPassword` as the reset callback target.

## Reset password

- Page: `app/(auth)/reset-password/page.tsx`
- Form: `features/auth/components/password/reset-password-form.tsx`

The page reads `?token=` from the query string. When a token is present, the form posts `newPassword`, `confirmPassword`, and `token` to `resetPasswordAction`, which calls `auth.api.resetPassword`.

- Invalid or expired tokens return a normal form error.
- Successful reset shows inline confirmation plus a link back to `/sign-in`.

## Flow summary

1. User opens `/forgot-password` and submits an email.
2. Better Auth sends a reset link that lands on `/reset-password?token=...`.
3. User submits a new password.
4. The UI either shows token error or success with a sign-in link.
