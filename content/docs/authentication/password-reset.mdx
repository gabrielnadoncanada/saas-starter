## Forgot password

- **Page:** `app/(auth)/forgot-password/page.tsx`
- **Form:** `features/auth/components/password/forgot-password-form.tsx`

The user submits their **email**. The form calls **`requestPasswordReset`** from `features/auth/client/auth-api-client.ts`:

```ts
export async function requestPasswordReset(email: string) {
  await authClient.requestPasswordReset({
    email: email.trim().toLowerCase(),
    redirectTo: routes.auth.resetPassword,
  });
}
```

After a successful request, the UI shows a neutral message: if an account exists, a reset link was sent. This avoids leaking whether the email is registered.

> **Note:** The toast and inline copy both use the same “if an account exists…” wording for consistency.

## Reset password (new password)

- **Page:** `app/(auth)/reset-password/page.tsx`
- **Form:** `features/auth/components/password/reset-password-form.tsx`

The page reads **`?token=`** from the query string. If the token is missing or blank, it shows a short message and a link back to **forgot password** instead of the form.

When `token` is present, **`ResetPasswordForm`** collects **new password** and **confirm** (schema: `features/auth/schemas/password-change.schema`) and calls **`resetPassword`** from `features/auth/client/auth-api-client.ts`. The client normalizes Better Auth errors:

```ts
// features/auth/client/auth-api-client.ts
export async function resetPassword(
  token: string,
  newPassword: string,
): Promise<ResetPasswordResult> {
  const { error } = await authClient.resetPassword({
    newPassword,
    token,
  });

  if (!error) {
    return { status: "success" };
  }

  if (getAuthErrorCode(error) === "INVALID_TOKEN") {
    return {
      status: "invalid_token",
      message: "This reset link is invalid or has expired.",
    };
  }

  const message = getAuthErrorMessage(
    error,
    "Unable to reset password. Please try again.",
  );
  return { status: "error", message };
}
```

The form sets **`errors.root`** when `result.status !== "success"`; on success it shows inline message plus **Go to sign in** to `routes.auth.login` (`/sign-in`).

## End-to-end flow

1. User opens **Forgot password**, submits email.
2. Email contains a link to **`/reset-password?token=...`** (via `redirectTo: routes.auth.resetPassword`).
3. User sets a new password; invalid or expired tokens show an error from the **`invalid_token`** branch.

> **Warning:** Reset links are single-use and time-limited by Better Auth; expired links require requesting a new email from **Forgot password**.
