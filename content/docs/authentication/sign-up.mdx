## Where it lives

- Page: `app/(auth)/sign-up/page.tsx`
- Form: `features/auth/components/sign-up-form.tsx`
- Public actions: `features/auth/actions/public-auth.actions.ts`
- Post-registration page: `app/(auth)/verify-email/sent/page.tsx`

## Email and password registration

The sign-up form is a single `useActionState` form. It posts `email`, `password`, `confirmPassword`, and `callbackUrl` to `signUpAction`, which calls `auth.api.signUpEmail`.

- Field validation comes from the Zod schema.
- Success redirects to `/verify-email/sent`, preserving `callbackUrl`.
- If Better Auth returns an explicit duplicate-email error, the action maps it to the `email` field.

## Email verification

The starter requires email verification before normal password sign-in. After sign-up, users are sent to `/verify-email/sent`. If they later try to sign in before verification, the sign-in page can resend the verification email.

## Magic link and OAuth

The same secondary actions used on sign-in are available here:

- magic link via `authClient.signIn.magicLink`
- social auth via `authClient.signIn.social`

Magic link redirects to `/check-email` with the submitted email and preserved `callbackUrl`.
