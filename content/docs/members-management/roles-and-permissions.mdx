## Organization roles

Canonical workspace roles are defined as string literals in `shared/lib/db/enums.ts`:

```ts
export const ORG_ROLES = ["owner", "admin", "member"] as const;
export type OrgRole = (typeof ORG_ROLES)[number];
```

They are stored as **plain strings** on `Member.role` (and invitation `role`) in the database, matching the Better Auth organization plugin—not Prisma enums.

## Parsing and primary role

`shared/lib/auth/roles.ts` provides generic helpers:

- **`parseRoles(value)`** — accepts a string or array; splits comma-separated values and trims.
- **`hasRole` / `hasAnyRole`** — membership checks on parsed roles.
- **`getPrimaryRole(parsed, priorities, fallback)`** — returns the first match from an ordered list.

`shared/lib/db/enums.ts` wraps them for orgs with **`parseOrgRoles`**, **`hasOrgRole`**, **`getPrimaryOrgRole`** (default fallback **`member`**), using priority order **`owner` → `admin` → `member`** so the UI shows one badge when multiple strings are present.

## What each role means in this starter

| Role    | Typical meaning |
|---------|------------------|
| `owner` | Full workspace control in this codebase: rename/delete org, invite/remove members, invitation management UI. Set for the creator (`creatorRole: "owner"` in `auth-config.ts`). |
| `admin` | Available for invitations; **no separate admin-only UI** is implemented beyond being a member with a higher label—server actions for org management still require **owner**. |
| `member` | Default invited role; standard member access. |

> **Note:** If you need admins to manage members, extend `requireActiveOrganizationRole` call sites and `canManageMembers` props to include `"admin"`—the starter does not do that today.

## Platform admin (separate from org roles)

```ts
export const PLATFORM_ADMIN_ROLES = ["admin"] as const;

export function isPlatformAdmin(value: unknown): boolean {
  return hasAnyRole(value, PLATFORM_ADMIN_ROLES);
}
```

This applies to **`user.role`** on the account (Better Auth **admin** plugin), used e.g. in `DashboardSidebarUser` to show the Admin link. It is **not** the same as `Member.role` inside an organization.
