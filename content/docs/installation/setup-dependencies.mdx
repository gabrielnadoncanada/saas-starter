## Install packages

From the repository root:

```bash
pnpm install
```

This resolves dependencies declared in `package.json` and runs the **`postinstall`** script, which executes Prisma client generation automatically:

```json
"postinstall": "pnpm exec prisma generate"
```

So you normally **do not** need to run `pnpm db:generate` immediately after a fresh install unless something failed during `postinstall`.

## Regenerate the Prisma client manually

If you change the Prisma schema or suspect the client is stale:

```bash
pnpm db:generate
```

This runs `pnpm exec prisma generate` as defined in `package.json`.

## Related scripts (reference)

From `package.json`:

| Script        | Command                          |
| ------------- | -------------------------------- |
| `db:generate` | `pnpm exec prisma generate`      |
| `db:format`   | `pnpm exec prisma format`        |

## Prisma config

The datasource URL is read from `POSTGRES_URL` via `prisma.config.ts` (not from a single `schema.prisma` `url` field). Ensure your environment is loaded before running Prisma CLI commands that touch the database (see [Project setup](./project-setup)).

> **Warning:** Use **pnpm** for this repo. Mixing `npm install` or `yarn` can produce inconsistent lockfiles and duplicate dependencies.
