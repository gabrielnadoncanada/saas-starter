This page documents the variables in **`.env.example`** as they exist in the codebase. Copy that file to `.env` and fill in real values for your environment.

## Database

| Variable       | Purpose |
| -------------- | ------- |
| `POSTGRES_URL` | PostgreSQL connection string used by Prisma (`prisma.config.ts`) and the app (`shared/lib/db/prisma.ts`). |

Example shape (placeholders):

```env
POSTGRES_URL=postgresql://***
```

## App URLs and auth

| Variable               | Purpose |
| ---------------------- | ------- |
| `BASE_URL`             | Canonical app URL (e.g. `http://localhost:3000` locally). Used for auth callbacks and absolute links. |
| `AUTH_SECRET`          | Secret for Better Auth (signing sessions and tokens). Must be a strong random value in production. |
| `BETTER_AUTH_API_KEY`  | Optional API key if you enable Better Auth API routes that require it (can stay empty if unused). |

```env
BASE_URL=http://localhost:3000
AUTH_SECRET=***
BETTER_AUTH_API_KEY=
```

## Stripe

### Core and webhooks

| Variable                 | Purpose |
| ------------------------ | ------- |
| `STRIPE_SECRET_KEY`      | Stripe secret API key (`sk_test_...` or `sk_live_...`). |
| `STRIPE_WEBHOOK_SECRET`  | Webhook signing secret from the Stripe Dashboard or Stripe CLI (`whsec_...`). |

### Price IDs (paid plans)

These four are **required** for paid plans to appear in billing and pricing UI (per comment in `.env.example`):

| Variable                    | Purpose |
| --------------------------- | ------- |
| `STRIPE_PRICE_PRO_MONTHLY`  | Stripe Price ID for Pro monthly. |
| `STRIPE_PRICE_PRO_YEARLY`   | Stripe Price ID for Pro yearly. |
| `STRIPE_PRICE_TEAM_MONTHLY` | Stripe Price ID for Team monthly. |
| `STRIPE_PRICE_TEAM_YEARLY`  | Stripe Price ID for Team yearly. |

### Billing behavior

| Variable                           | Purpose |
| ---------------------------------- | ------- |
| `STRIPE_ENABLE_TRIAL_WITHOUT_CC`   | Set subscription trials without collecting a card when supported by your Stripe setup. |
| `ENABLE_AUTOMATIC_TAX_CALCULATION` | Toggle automatic tax calculation in Stripe Checkout / Customer Portal flows. |
| `ENABLE_TAX_ID_COLLECTION`         | Toggle tax ID collection during checkout. |
| `STRIPE_REQUIRE_BILLING_ADDRESS`   | Require a billing address when applicable. |

## Email (Resend)

| Variable         | Purpose |
| ---------------- | ------- |
| `RESEND_API_KEY` | Resend API key for sending transactional email. |
| `EMAIL_FROM`     | Default “from” address (e.g. `Acme <hello@example.com>`). |
| `EMAIL_REPLY_TO` | Optional reply-to address. |

```env
RESEND_API_KEY=re_***
EMAIL_FROM=Acme <hello@example.com>
EMAIL_REPLY_TO=
```

## OAuth

| Variable                   | Purpose |
| -------------------------- | ------- |
| `GOOGLE_CLIENT_ID`         | Google OAuth client ID (optional if you only use email/password). |
| `GOOGLE_CLIENT_SECRET`     | Google OAuth client secret. |
| `GITHUB_CLIENT_ID`         | GitHub OAuth client ID. |
| `GITHUB_CLIENT_SECRET`     | GitHub OAuth client secret. |
| `ALLOW_EMAIL_ACCOUNT_LINKING` | Set to `"true"` to enable automatic OAuth account linking by email when providers verify email (Google and GitHub do). |

## Public product configuration

| Variable                   | Purpose |
| -------------------------- | ------- |
| `NEXT_PUBLIC_ACCOUNT_MODE` | `"personal-only"` or `"organizations-only"`. Defaults to **`organizations-only`** if unset (per `.env.example`). |

> **Warning:** Never commit real `.env` files or live secrets. Keep secrets in a password manager or your host’s secret store.
