This page complements `.env.example` and [Environment variables](../installation/environment-variables.mdx). Values below are **categories**, not copy-paste secrets.

## What usually changes from dev to prod

| Area                        | Development                                     | Production                                  |
| --------------------------- | ----------------------------------------------- | ------------------------------------------- |
| **`BASE_URL`**              | `http://localhost:3000`                         | `https://your-domain.com`                   |
| **`POSTGRES_URL`**          | Local or shared dev DB                          | Managed Postgres with TLS and backups       |
| **`AUTH_SECRET`**           | Any strong random string                        | **New** long random string; never reuse dev |
| **Stripe keys**             | `sk_test_...`, test price IDs                   | `sk_live_...`, **live** price IDs           |
| **`STRIPE_WEBHOOK_SECRET`** | From `stripe listen` or Dashboard test endpoint | From **live** webhook endpoint              |
| **Resend**                  | Sandbox/testing sender                          | Verified domain, production API key         |
| **OAuth**                   | `http://localhost:3000/...` redirect URIs       | Production `BASE_URL` redirect URIs         |

> **Note:** `NEXT_PUBLIC_*` variables are embedded at build time. Change them before running `pnpm build` if they must differ per environment.

## Database

```
POSTGRES_URL=postgresql://...
```

Production should use a connection string from your host (SSL parameters as required). The app uses Prisma with this URL per `shared/lib/db/prisma.ts` and `prisma.config.ts`.

## App URLs and auth

From `.env.example`:

```
# App URLs and auth
BASE_URL=http://localhost:3000
AUTH_SECRET=***
BETTER_AUTH_API_KEY=
```

- **`BASE_URL`** — Must match the URL users type in the browser (scheme + host + optional port).
- **`AUTH_SECRET`** — Required for Better Auth; treat like a root credential.
- **`BETTER_AUTH_API_KEY`** — Optional; only if you enable protected Better Auth API usage.

## Stripe

```
STRIPE_SECRET_KEY=sk_test_***
STRIPE_WEBHOOK_SECRET=whsec_***
```

Use **live** keys and **live** `STRIPE_PRICE_*` IDs in production. Webhook URL must be **`{BASE_URL}/api/stripe/webhook`** with the secret that signs events for that endpoint.

Billing behavior flags (`STRIPE_ENABLE_TRIAL_WITHOUT_CC`, tax, billing address) are product decisions—set explicitly for production.

## Email

```
RESEND_API_KEY=re_***
EMAIL_FROM=Acme <hello@example.com>
EMAIL_REPLY_TO=
```

Production **`EMAIL_FROM`** should use a domain verified in Resend to avoid deliverability issues.

## OAuth

```
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
ALLOW_EMAIL_ACCOUNT_LINKING=true
```

Provider consoles must list production redirect URLs derived from **`BASE_URL`**.

## Account mode

```
NEXT_PUBLIC_ACCOUNT_MODE=organizations-only
```

Same variable as in dev, but the chosen value affects UX for all users—confirm before shipping.

## Optional: S3 storage

Not in `.env.example`, but required together for S3-backed files:

- `STORAGE_S3_ENDPOINT`
- `STORAGE_S3_BUCKET`
- `STORAGE_S3_ACCESS_KEY_ID`
- `STORAGE_S3_SECRET_ACCESS_KEY`
- `STORAGE_S3_REGION`

If any is missing, the app uses **`.storage`** on disk (see [Storage configuration](../storage/configuration.mdx)).
