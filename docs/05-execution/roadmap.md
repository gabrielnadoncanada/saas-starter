# Roadmap

## Purpose
Define the phased execution plan from documentation setup to first sellable release.

## Phase 1 — Strategic foundation
### Goal
Clarify what is being sold, to whom, and why it wins.

### Deliverables
- documentation system initialized,
- vision defined,
- ICP defined,
- value proposition documented,
- MVP scope documented.

### Exit criteria
- clear positioning statement,
- clear buyer profile,
- clear V1 boundary.

## Phase 2 — Product packaging and architecture lock
### Goal
Freeze the high-level product direction and technical constraints before building too much.

### Deliverables
- pricing floor locked at 149 USD,
- structural standard locked to `nextjs-saas-structure`,
- buyer-fit standard locked to `starter-buyer-fit`,
- stack decisions narrowed,
- architecture principles finalized,
- package structure drafted.

### Risks
- drifting into over-engineering,
- building premium modules before validating core value.

## Phase 3 — Build the sellable core
### Goal
Implement the smallest product that feels complete and commercially credible at 149 USD and above.

### Deliverables
- auth,
- billing,
- dashboard shell,
- workspace foundation,
- settings,
- onboarding,
- landing page,
- setup docs,
- customization docs.

### Exit criteria
- local setup works from a clean machine,
- core flows are demoable,
- docs support self-serve usage,
- the product feels premium enough to defend the entry price.

## Phase 4 — Validation and refinement
### Goal
Pressure-test the offer with real buyers.

### Deliverables
- live demo,
- screenshots and landing page,
- buyer interviews or feedback loops,
- revised messaging based on objections.

### Success indicators
- buyers understand the offer quickly,
- objections are predictable,
- customization confidence is high,
- pricing resistance at 149 USD is manageable.

## Phase 5 — Launch
### Goal
Convert interest into first sales and real-world learning.

### Deliverables
- launch page,
- payment flow,
- documentation portal or docs directory,
- launch content,
- post-purchase onboarding.

## Near-term critical path
1. finalize exact V1 blueprint,
2. lock stack and billing path,
3. define Starter vs Pro split,
4. build core monetization-ready modules,
5. package proof and messaging.
