# Buyer Persona

## Core Buyer
The starter is designed for:
- solo founders
- consultants and freelancers shipping SaaS quickly
- indie hackers
- two-to-five person technical teams

## Buyer Goals
The buyer is purchasing:
- a fast path to a launchable SaaS
- fewer hours lost on auth, billing, settings, and common foundation work
- a codebase they can understand quickly
- a starter they can modify without fear
- visible product quality that helps them ship faster

## Buyer Expectations
The buyer expects:
- obvious structure
- predictable conventions
- minimal ceremony
- direct customization paths
- production-credible UI
- documentation that helps them change things fast

## Non-Buyer
The starter is not optimized for:
- enterprise architecture purists
- teams wanting strict clean architecture or DDD by default
- buyers willing to learn a framework-within-a-framework before editing features

## Buying Trigger
The buyer should be able to think:
"I can launch with this quickly, and I will not regret buying it."
