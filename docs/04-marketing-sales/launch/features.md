# Features

## Purpose
List the features that can be sold clearly and frame them in buyer language.

## Saleable feature inventory for launch

### 1. Modern stack foundation
What the buyer gets:
- Next.js 16 baseline
- React 19 baseline
- Prisma/Postgres data layer baseline
- Tailwind 4 styling baseline
- Stripe-ready billing integration direction

Buyer value:
- starts from current tooling instead of dated boilerplate
- reduces early stack decisions
- improves credibility for product launches

### 2. Authentication foundation
What the buyer gets:
- authentication setup already wired into the app
- provider architecture in place
- protected route baseline

Buyer value:
- avoids rebuilding login/session plumbing
- gives a real starting point instead of a fake template auth screen

### 3. Protected dashboard foundation
What the buyer gets:
- dashboard route protection
- authenticated application shell direction
- basis for product pages after login

Buyer value:
- starts from a real app shape, not just a marketing site
- easier to extend into product modules

### 4. Billing foundation
What the buyer gets:
- Stripe integration direction
- billing groundwork that supports SaaS monetization
- clearer path to subscriptions than starting from zero

Buyer value:
- avoids one of the most annoying parts of SaaS setup
- makes the product feel monetization-oriented

### 5. Database and local setup baseline
What the buyer gets:
- Prisma setup
- migration and seed workflow
- scripts for local database lifecycle

Buyer value:
- lower setup anxiety
- easier local onboarding
- more confidence that the starter is serious

### 6. Pragmatic code organization
What the buyer gets:
- structure aligned with the project's architecture standards
- feature ownership that is easier to follow than overbuilt abstractions
- clearer path for editing and extension

Buyer value:
- faster time to understand
- faster time to modify
- lower cognitive overhead than enterprise-leaning starters

## Claims to avoid until fully confirmed
- full workspace management
- complete team system
- advanced RBAC
- deep AI functionality as a core launch promise
- exhaustive admin capabilities

## Feature section pattern for the landing page
For each feature block:
1. what is included
2. why it matters
3. how it saves time or reduces risk

## Proof strategy
Use screenshots, code snippets, setup docs, and one short demo video to make the features feel real.
