# Launch Checklist

## Purpose
Track the minimum work required to launch with confidence.

## Offer and positioning
- [ ] headline finalized
- [ ] subheadline finalized
- [ ] Starter / Pro / Agency split finalized
- [ ] pricing page copy finalized
- [ ] licensing terms written clearly
- [ ] refund policy decided

## Product truth validation
- [ ] billing flows audited and documented
- [ ] auth flows audited and documented
- [ ] settings/account scope confirmed
- [ ] workspace/team scope confirmed or removed from copy
- [ ] feature list aligned with actual shipped code

## Sales assets
- [ ] landing page written
- [ ] pricing page written
- [ ] comparison section written
- [ ] screenshots captured
- [ ] short demo video recorded
- [ ] product thumbnail / hero visual prepared

## Documentation
- [ ] public README rewritten for buyers
- [ ] setup guide validated from a clean environment
- [ ] customization guide written
- [ ] environment variable guide written
- [ ] billing setup guide written

## Delivery and operations
- [ ] checkout flow selected
- [ ] post-purchase delivery flow selected
- [ ] support contact method defined
- [ ] update policy defined
- [ ] analytics installed on landing page

## Launch distribution
- [ ] X launch thread drafted
- [ ] Reddit launch post drafted
- [ ] LinkedIn post drafted
- [ ] product communities list prepared
- [ ] first 10 warm outreach targets listed

## Final rule
Do not launch with copy that is more polished than the truth of the product.
