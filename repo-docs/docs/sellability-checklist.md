# Sellability Checklist

Use this before selling the starter.

## Buyer Fit
- [ ] The product clearly targets solo founders, consultants, or small technical teams.
- [ ] The codebase does not feel enterprise-heavy.
- [ ] A buyer can modify common flows without learning custom internal patterns.

## Core Product Credibility
- [ ] Auth feels production-ready.
- [ ] Billing feels real, not demo-level.
- [ ] Plans, feature gating, and usage limits are easy to extend.
- [ ] Dashboard feels credible and not placeholder-like.
- [ ] Settings are complete enough to feel launch-ready.

## Modification Speed
- [ ] Adding a new entity is straightforward.
- [ ] Editing a current flow is straightforward.
- [ ] File ownership is obvious.
- [ ] Naming is predictable.
- [ ] Simple work does not require too many files.

## Cognitive Simplicity
- [ ] No unnecessary repositories, handlers, or use-cases in ordinary feature work.
- [ ] No framework-within-a-framework drift.
- [ ] Helpers and wrappers are obvious and justified.

## Perceived Value
- [ ] The starter feels worth the price on first inspection.
- [ ] The UI creates confidence quickly.
- [ ] The codebase looks easier to work with than competitors that feel heavy.
- [ ] The product has a clear angle or differentiation.

## Docs Readiness
- [ ] Setup is documented.
- [ ] Customization is documented.
- [ ] Common extension paths are documented.
- [ ] Iteration history or current priorities are documented.
